# 第 7 题：困惑度（Perplexity）如何评估语言模型

## 题目

什么是困惑度（Perplexity）？它如何评估语言模型性能？

---

## 一、定义与公式

**困惑度（Perplexity, PPL）** 是语言模型在**测试集**上对“下一个 token 预测”的**平均不确定程度**的度量，等价于**交叉熵的指数**。

设测试序列为 $x_1, x_2, \ldots, x_N$（$N$ 为 token 总数），模型给出 $P(x_i \mid x_{<i})$，则：

$$
\mathrm{PPL} = \exp\left( -\frac{1}{N} \sum_{i=1}^{N} \ln P(x_i \mid x_{<i}) \right) = \exp(\overline{\mathrm{CE}})
$$

其中 $\overline{\mathrm{CE}}$ 就是**平均交叉熵**（每个 token 的负对数似然的平均）。所以：**PPL 越低 = 平均交叉熵越低 = 模型对真实下一个 token 给的概率越高 = 语言建模越好**。

---

## 二、直观含义

- **PPL = 1**：理想情况，模型对每个真实 token 都赋概率 1，即“完全不困惑”。  
- **PPL 越大**：模型越“困惑”，对真实 token 赋的概率越低，预测越差。  
- **等价解释**：PPL 可以理解为“模型觉得下一个 token 平均有多少个等概选项”。例如 PPL = 100，可粗略理解为平均在约 100 个等概选项中犹豫。

**面试可能问**：“PPL 和 loss 的关系？”——PPL = exp(loss)，若 loss 是平均交叉熵（对自然对数），则两者一一对应；PPL 下降等价于 loss 下降。

---

## 三、怎么用来评估

- 在**留出测试集**上，用模型计算每个位置 $i$ 的 $P(x_i \mid x_{<i})$，按上式对**所有 token** 求平均再取指数，得到 token 级 PPL。  
- 有时也按**句**或**文档**先算每句/每文档的 PPL，再对句/文档平均，便于分析不同领域或长度。  
- **用途**：和 baseline（更小模型、不同架构、不同数据）对比；PPL 下降一般表示语言建模能力提升。

**注意**：PPL 主要反映**续写/生成**的流畅度和一致性，和**下游任务**（问答、推理、分类）的相关性不一定强。有的模型 PPL 略高但下游更好（因为学了更多“有用”的表示）。所以评估时往往要**既看 PPL，也看下游指标**。

---

## 四、小结与面试要点

**小结**：困惑度 = 测试集上平均交叉熵的指数，越低表示模型对真实 token 越自信、语言建模越好；通过留出测试集、token 级平均负对数似然再取 exp 得到。

**面试要点**：

- 公式：PPL = exp(平均交叉熵) = exp(-1/N Σ ln P(x_i|x_{<i}))。  
- 直观：越低越好；可理解为“平均等概选项数”。  
- 和 loss 关系：PPL = exp(loss)（当 loss 为平均 CE 时）。  
- 局限：主要反映语言建模，与下游任务不完全一致，要结合下游指标看。

---

## 记忆要点

1. PPL = exp(平均交叉熵)，越低语言建模越好。  
2. 直观：“平均等概选项数”；PPL=1 为理想。  
3. 计算：留出测试集，token 级平均 -ln P(x_i|x_{<i})，再取 exp。  
4. 与 loss：PPL = exp(loss)。  
5. 局限：主要反映续写能力，下游任务需单独看指标。
