# 第 5 题：KV Cache 的原理与在自回归生成中的作用

## 题目

解释 KV Cache 的原理，以及它在自回归生成中的作用。

---

## 一、自回归生成时在算什么？

自回归生成是**一个 token 一个 token**地生成：第 $t$ 步只多一个**新 token**，对应一个**新的 query** $q_t$；而 key 和 value 来自**当前及之前所有 token**（1 到 $t$）的表示。若**不做缓存**，每步都要对“从 1 到 t”的整段序列重新前向一遍，那么第 $t$ 步会重复计算 1 到 $t-1$ 的 K、V，浪费大量计算。

**不缓存的复杂度**：第 $t$ 步要算 $t$ 个 token 的 K/V 和一次 Attention，总步数若为 $L$，K/V 的计算量是 $O(1+2+\cdots+L) = O(L^2)$，非常贵。

---

## 二、KV Cache 做了什么？

**KV Cache** 的思路：把**已经算过的**每个位置的 Key 和 Value **存下来**，生成下一步时：

- 只对**当前新 token** 算一次前向，得到 $q_t, k_t, v_t$；  
- 把 $k_t, v_t$ **拼到**已有的 K、V 缓存后面；  
- 用 $q_t$ 和**整段缓存的** $K_{1:t}, V_{1:t}$ 做 Attention，得到当前步的输出。

这样每步**只算 1 个**新 token 的 K/V，其余都用缓存，避免对历史 token 重复计算。

**形式化**：设已生成 $t-1$ 个 token，缓存为 $K_{1:t-1}, V_{1:t-1}$。第 $t$ 步：

1. 新 token 的隐状态经过当前层 → 得到 $q_t, k_t, v_t$。  
2. 更新缓存：$K_{1:t} = [K_{1:t-1}; k_t]$，$V_{1:t} = [V_{1:t-1}; v_t]$（按序列维拼接）。  
3. $\mathrm{Attention}(q_t, K_{1:t}, V_{1:t})$ 得到当前层的输出。

**多层时**：每一层都有自己的 K、V 缓存，每步每层都只算新 token 的 $k_t, v_t$ 并拼到该层缓存，再和 $q_t$ 做 Attention。

---

## 三、作用与代价

**作用**：

- **计算**：每步 K/V 计算从 $O(t)$ 变成 $O(1)$（只算新 token），总 K/V 计算从 $O(L^2)$ 降到 $O(L)$，解码明显加速。  
- **延迟**：每步只跑一次小前向，首 token 延迟和每 token 延迟都能控制得更好。

**代价**：

- **显存**：要存所有层、所有已生成位置的 K 和 V。设层数为 $L$，头数为 $h$，头维度 $d_k$，序列长 $n$，batch $B$，则 KV Cache 的显存大约是 $2 \cdot L \cdot B \cdot n \cdot h \cdot d_k \cdot \mathrm{sizeof}(\mathrm{dtype})$，和**序列长度、层数、batch**都线性相关。长序列、大 batch 时显存压力很大。  
- **优化手段**：限制最大生成长度、减小 batch、使用 PagedAttention（把 KV 分页管理）等，都是为了在长序列下还能跑得动。

面试常问：“KV Cache 占多少显存？”——按上面公式估算；或问“为什么长序列推理费显存？”——因为 Cache 随长度线性增长，且层数、batch 都会乘上去。

---

## 四、小结与面试要点

**小结**：KV Cache 缓存历史 token 的 K、V，每步只算新 token 的 K/V 并拼到缓存，再做 Attention；大幅减少重复计算、加速解码，代价是显存随序列长度、层数、batch 线性增长。

**面试要点**：

- 原理：缓存已算过的 K、V，每步只算 $k_t, v_t$ 并拼到缓存，用 $q_t$ 和整段 K、V 做 Attention。  
- 作用：避免重复算历史 K/V，解码从每步 O(t) 降到 O(1)，总 O(L²) 降到 O(L)。  
- 代价：显存存所有层、所有位置的 K 和 V，与长度、层数、batch 线性相关。  
- 长序列怎么办？限制长度、减 batch、PagedAttention 等。

---

## 记忆要点

1. KV Cache：缓存已算过的 K、V，每步只算新 token 的 $k_t,v_t$ 并拼到缓存。  
2. 作用：解码时 K/V 从每步 O(t) 变为 O(1)，总计算从 O(L²) 到 O(L)。  
3. 代价：显存与序列长度、层数、batch 线性相关。  
4. 长序列优化：限制长度、减小 batch、PagedAttention。
