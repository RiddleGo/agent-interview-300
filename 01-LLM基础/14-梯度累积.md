# 第 14 题：梯度累积如何解决显存不足

## 题目

什么是梯度累积？它如何解决显存不足问题？

---

## 一、什么是梯度累积？

**梯度累积（Gradient Accumulation）** 指把一次“逻辑上的大 batch”拆成多个**小 mini-batch** 依次前向、反向，但**不立即更新参数**，而是把每次得到的梯度**累加**（或平均），在累加够若干步后再用累加/平均后的梯度做一次参数更新。

等价于：用多个小 batch 的梯度平均，模拟“大 batch”的梯度，再更新一次。

---

## 二、如何缓解显存不足？

- **显存瓶颈**：大 batch 需要同时存更多激活和梯度，容易 OOM。  
- **做法**：batch size 减半（或缩小为 1/k），每次只算一个小 batch 的梯度并累加到缓冲区；累加 k 次后，用（累加梯度 / k）更新参数，然后清空梯度、继续下一轮累积。  
- **效果**：单次前向/反向的显存占用由“小 batch”决定，因此**显存需求下降**；而有效 batch size = 小 batch × 累积步数，**等效大 batch 训练**，有利于稳定与收敛。

---

## 三、注意点

- 若使用 BatchNorm，BN 的统计量仍是每个小 batch 的，与“真·大 batch”的 BN 不完全一致；对 LN 或没有 BN 的模型无此问题。  
- 累积步数过多时，等效 batch 很大，可能需相应调大学习率或 warmup；且每 k 步才更新一次，总 step 数会少，有时需要更多 epoch 或更多迭代次数来补偿。

**面试常问**：“和直接开大 batch 有什么区别？”——显存上等价于小 batch，所以能跑；有效 batch 变大，梯度更稳。 “有 BN 怎么办？”——BN 的统计量仍是每个小 batch 的，和“真·大 batch”的 BN 不完全一致；Transformer 用 LN 没有这个问题。

---

## 四、小结与面试要点

**小结**：梯度累积 = 多步前向反向、梯度累加（或平均），每 k 步更新一次；有效 batch = 小 batch × k，单步显存由小 batch 决定，缓解 OOM。

**面试要点**：做法（多步小 batch、梯度累加、每 k 步更新）；作用（等效大 batch、省显存）；注意（BN 仍按小 batch；k 大时可能要调学习率）。

---

## 记忆要点

1. 梯度累积：多步小 batch 梯度累加/平均，每 k 步更新；有效 batch = 小 batch × k。  
2. 作用：单步显存由小 batch 决定，缓解 OOM；等效大 batch 训练。  
3. 注意：BN 统计按小 batch；k 大时需调学习率或总迭代。
