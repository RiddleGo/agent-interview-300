## 序章：明天要面「Agent 全栈」，今晚只有一夜

你收到 HR 的消息：面试范围是 **Agent 算法工程师 / LLM 应用全栈**，请系统准备。

「Agent 全栈」听起来很帅，意思就是：从大模型怎么算、怎么训，到 Agent 怎么思考、怎么用工具、怎么和 RAG 搭在一起，再到怎么部署、怎么守安全合规——中间每一环都可能问到。所以范围特别大。

你打开那份 **300 题** 清单——从 [Self-Attention 机制](01-LLM基础/01-Self-Attention机制.md) 到 [总结与展望](13-行业应用/300-总结与展望.md)，从 [ReAct 框架](02-Agent核心概念/24-ReAct.md) 到 [AI 治理与伦理总结](11-AI治理与伦理/250-AI治理与伦理总结.md)——密密麻麻，像一张从「模型原理」画到「行业落地」的长卷。

说白了：前面是「大模型凭什么能推理、能对话」，中间是「Agent 怎么用工具、怎么规划、怎么和知识库结合」，后面是「怎么工程化、怎么安全、怎么合规」。你心想：这哪是 300 题，这是 300 个「和某某有什么区别」「你们怎么设计的」「手写/口述一个」的连环拷问。光背题号容易忘，不如把整条链路串成一条线，记故事比记列表轻松。

与其对着题号硬啃，不如换一种玩法：**把自己当成「从零搭建一个 Agent」**，从早晨打地基到入夜守底线，经历一遍「模型 → Agent 能力 → 提示 → 单体/多体 → RAG → 框架与集成 → 性能 → 安全与伦理 → 趋势与行业」的完整一天。

- **早晨** 先打地基：**LLM 基础**（Transformer、训练推理、量化）
- **上午** 学「会思考、会动手」：**Agent 核心概念** + **提示工程**
- **中午** 搭骨架：**单 Agent 架构**（工具、记忆、规划）+ **多 Agent 系统**
- **下午** 接外脑：**RAG 与知识增强**
- **傍晚** 上工程：**开发框架与工具**、**系统集成**、**性能优化**
- **入夜** 守底线：**安全**、**AI 治理与伦理**
- **尾声** 看远方：**最新技术趋势**、**行业应用**

三百个知识点，都藏在这条线里；读到哪，点到哪，跳过去就是那道题的详解。不用一次看完，当闲书翻也行。

准备好了？我们开始。

## 一、早晨：地基——LLM 凭什么能「想」和「说」

Agent 的「大脑」是大模型。模型不会算、不会推，后面工具调用、规划、RAG 全是空中楼阁。所以第一块一定是 **LLM 基础**：模型怎么表示序列、怎么训练、怎么推理、怎么省算力。

你想想：面试官问「Agent 和纯 LLM 有啥区别」，你要是连 [Self-Attention](01-LLM基础/01-Self-Attention机制.md)、[KV Cache](01-LLM基础/05-KV-Cache.md)、[解码策略](01-LLM基础/06-解码策略.md) 都说不清，就很难往上聊。**专业上**：Attention 的复杂度、位置编码选型、推理时的 Cache 和采样策略，都是高频题。

**表示与结构**：[位置编码](01-LLM基础/02-位置编码.md)（绝对 vs 相对）、[Layer Norm 及为何不用 BN](01-LLM基础/03-Layer-Normalization.md)、[Attention Mask](01-LLM基础/04-Attention-Mask.md)、[RoPE](01-LLM基础/19-RoPE.md)、[SwiGLU](01-LLM基础/20-SwiGLU.md)、[GQA 与 MQA](01-LLM基础/18-GQA-MQA.md)——这些决定了模型「怎么看到上下文」「怎么省显存」，面试常考 RoPE 和 GQA 的动机。

**训练与推理**：[困惑度](01-LLM基础/07-Perplexity.md)、[涌现能力](01-LLM基础/08-涌现能力.md)、[上下文学习 ICL](01-LLM基础/09-上下文学习.md)、[Chain-of-Thought](01-LLM基础/10-Chain-of-Thought.md)、[三种架构对比](01-LLM基础/11-三种架构对比.md)；[并行](01-LLM基础/12-并行.md)、[混合精度](01-LLM基础/13-混合精度训练.md)、[梯度累积](01-LLM基础/14-梯度累积.md)；[LoRA](01-LLM基础/15-LoRA.md)、[微调方法对比](01-LLM基础/16-微调方法对比.md)、[模型量化](01-LLM基础/17-模型量化.md)。用人话说：CoT 和 ICL 是 Agent 能「一步步想」和「看例子就会」的基础；LoRA 和量化是落地时必谈的，面试会问「你们用全量微调还是 LoRA」「量化到 INT8 还是 INT4、掉点多少」。

早晨结束，地基打好了；下一步是搞清 **什么是 Agent、怎么和模型「说话」**。

## 二、上午（上）：灵魂——Agent 会思考、会用工具

**Agent 核心概念**要回答三件事：Agent 和纯 LLM 有啥区别、Agent 怎么「感知-决策-执行」、怎么用工具、怎么规划、怎么记事儿。就像人不仅会聊天，还会查日历、发邮件、拆任务——Agent 也要有「动手能力」和「多步推理能力」。

[AI Agent 与 LLM 区别](02-Agent核心概念/21-AI-Agent与LLM区别.md)、[PDA 循环](02-Agent核心概念/22-PDA循环.md)、[工具使用](02-Agent核心概念/23-工具使用.md)、[ReAct 框架](02-Agent核心概念/24-ReAct.md) 是必考四件套。用人话说：LLM 是「问一句答一句」，Agent 是「先想、再选工具、执行、看结果、再想」的循环；ReAct 就是把 Thought-Action-Observation 写进 prompt，面试官最爱问「ReAct 和 CoT 的区别」「你们用的什么 Agent 框架」。

[多 Agent 系统](02-Agent核心概念/26-多Agent系统.md)、[记忆机制](02-Agent核心概念/27-记忆机制.md)、[规划](02-Agent核心概念/28-规划.md)、[函数调用](02-Agent核心概念/30-函数调用.md)、[自我反思](02-Agent核心概念/31-自我反思.md)、[目标分解](02-Agent核心概念/34-目标分解.md)、[人机协作](02-Agent核心概念/35-人机协作.md)、[世界模型](02-Agent核心概念/37-世界模型.md)、[Agent 幻觉](02-Agent核心概念/40-Agent幻觉.md)：记忆分短期（当前对话）和长期（知识库/向量）；规划是单步反应 vs 多步拆任务；函数调用是「模型输出结构化调用、后端执行」；自我反思是「错了再试」；幻觉是 Agent 乱编工具结果或乱推理，要缓解。**面试一句「你们 Agent 有没有反思/规划」就能扯出一串**，建议把 ReAct、工具调用、记忆、规划各准备一个简洁版说法。

## 三、上午（下）：表达——怎么和模型「说」才管用

**提示工程**是「和模型说话」的学问。同样的能力，提示写不好，效果差一截。所以从 [提示工程核心原则](03-提示工程/41-提示工程.md)、[Zero/One/Few-shot](03-提示工程/42-Zero-One-Few-shot.md)、[角色提示](03-提示工程/43-角色提示.md) 到 [CoT 设计](03-提示工程/44-CoT设计模式.md)、[ReAct 提示模板](03-提示工程/49-ReAct提示模板.md)，都是让模型「听懂任务、按格式输出」的基础。

[自一致性](03-提示工程/46-自一致性.md)、[Least-to-Most](03-提示工程/47-Least-to-Most.md)、[提示链](03-提示工程/48-提示链.md)、[自动提示优化](03-提示工程/50-自动提示优化.md)、[结构化提示](03-提示工程/52-结构化提示.md)、[上下文压缩](03-提示工程/53-上下文压缩.md) 帮你省 token、提稳定；[提示注入](03-提示工程/54-提示注入.md)、[提示越狱](03-提示工程/55-提示越狱.md) 是安全侧必考——用人话说就是「用户输入里塞恶意指令、让模型忘掉系统设定」，防护手段要能说一两句。[提示评估与 A/B 测试](03-提示工程/60-提示评估与AB测试.md)、[动态提示与 RAG](03-提示工程/58-动态提示与RAG.md) 是落地时常谈的。

上午结束，你已经有「会想、会动手、会说话」的灵魂和表达；接下来是 **一个 Agent 怎么搭、多个 Agent 怎么协作**。

## 四、中午（上）：单体——一个 Agent 的完整骨架

**单 Agent 架构**把「工具、记忆、规划、反思」落成具体设计：怎么选工具、怎么规划多步、怎么存历史、错了怎么恢复、怎么观测和运维。

## 五、中午（下）：协作——多 Agent 怎么一起干活

**多 Agent 系统**要解决：任务怎么拆、谁来做、结果怎么汇总、冲突怎么解决、怎么容错、怎么编排。就像一个小团队，有人查资料、有人写代码、有人审——多 Agent 就是多个「小模型/能力」分工协作。

[CrewAI、AutoGen、MetaGPT](05-多Agent系统/87-CrewAI-AutoGen-MetaGPT.md) 是框架常考；[任务委托与结果汇总](05-多Agent系统/88-任务委托与结果汇总.md)、[冲突解决](05-多Agent系统/89-冲突解决机制.md)、[共识机制](05-多Agent系统/90-共识机制.md)、[负载均衡](05-多Agent系统/91-负载均衡.md)、[容错机制](05-多Agent系统/92-容错机制.md)、[层级 vs 扁平架构](05-多Agent系统/93-层级与扁平架构.md)、[编排 vs 编舞](05-多Agent系统/102-编排与编舞.md) 是设计题。用人话说：编排是「有一个中心指挥」，编舞是「各 Agent 按协议自发配合」；面试会问「你们多 Agent 怎么分工、怎么防互相抢活或死锁」。

[权限与访问控制](05-多Agent系统/100-权限与访问控制.md)、[安全隔离](05-多Agent系统/110-安全隔离机制.md)、[日志聚合与链路追踪](05-多Agent系统/107-日志聚合与链路追踪.md)、[成本优化](05-多Agent系统/104-成本优化策略.md)、[优雅关闭与状态持久化](05-多Agent系统/103-优雅关闭与状态持久化.md) 是上线和运维必谈。

中午过完，单体和多体骨架都齐了；下午轮到 **给 Agent 接「外脑」——RAG**。

## 六、下午：外脑——RAG 与知识增强

Agent 不能只靠参数里的知识，要能查文档、查数据库。**RAG（检索增强生成）**就是「先检索相关片段，再塞进 prompt 让模型生成」，这样模型就能「站在你的知识库上」回答。

[RAG 基础与流程](06-RAG与知识增强/111-RAG基础.md)、[检索策略](06-RAG与知识增强/112-检索策略.md)（稠密/稀疏/混合）、[分块策略](06-RAG与知识增强/113-分块策略.md)、[向量数据库选型](06-RAG与知识增强/114-向量数据库选型.md)、[重排序 Reranker](06-RAG与知识增强/115-重排序.md)、[查询改写与扩展](06-RAG与知识增强/116-查询改写与扩展.md) 是 RAG 六件套。用人话说：分块决定「一段多长、按句还是按段」；检索决定「用向量相似度还是关键词还是混着来」；Reranker 是「粗召回后再精排」，面试常考「你们 RAG 几路召回、有没有 Reranker」。

[GraphRAG 与知识图谱](06-RAG与知识增强/117-GraphRAG与知识图谱.md)、[多路召回与融合](06-RAG与知识增强/118-多路召回与融合.md)、[RAG 评估指标](06-RAG与知识增强/119-RAG评估指标.md)、[RAG 幻觉与引用溯源](06-RAG与知识增强/120-RAG幻觉与引用.md)、[RAG 与 Agent 结合](06-RAG与知识增强/124-RAG与Agent结合.md)、[RAG vs 微调](06-RAG与知识增强/126-RAG与微调对比.md)：GraphRAG 用图结构做多跳推理；RAG 幻觉是「模型引了错段落或瞎编」，要引用溯源和评估；和 Agent 结合时往往是「Agent 先决定查什么、再调 RAG」。**面试一句「你们 RAG 和 Agent 怎么配合」就能展开**，建议准备「谁调谁、检索条件谁生成」的说法。

下午结束，Agent 已经能「带知识库干活」；傍晚是 **用啥写、怎么接系统、怎么跑得快**。

## 九、入夜：底线——安全、治理与伦理

Agent 能调工具、能执行代码、能访问数据，所以**安全**必须跟上：提示注入、越狱、权限、审计、数据隐私、密钥管理、合规。**AI 治理与伦理**要回答：模型对齐没有、可解释吗、出事了谁负责、公平性偏见怎么控。

[提示注入与防护](10-安全/211-提示注入与防护.md)、[权限与访问控制](10-安全/212-权限与访问控制.md)、[数据安全与隐私](10-安全/213-数据安全与隐私.md)、[审计与合规](10-安全/214-审计与合规.md)、[安全沙箱](10-安全/215-安全沙箱.md)、[密钥与凭证管理](10-安全/217-密钥与凭证管理.md)、[安全开发生命周期](10-安全/218-安全开发生命周期.md)、[内容安全与审核](10-安全/234-内容安全与审核.md)、[安全最佳实践](10-安全/235-安全最佳实践.md)：提示注入是「用户输入里夹带指令、篡改系统行为」，防护手段（过滤、隔离、最小权限）要能说；面试会问「你们怎么防注入、怎么管权限」。

[AI 对齐概述](11-AI治理与伦理/236-AI对齐概述.md)、[RLHF 与人类反馈](11-AI治理与伦理/237-RLHF与人类反馈.md)、[可解释性与可审计](11-AI治理与伦理/238-可解释性与可审计.md)、[公平性与偏见](11-AI治理与伦理/239-公平性与偏见.md)、[问责与责任](11-AI治理与伦理/240-问责与责任.md)、[价值对齐](11-AI治理与伦理/241-价值对齐.md)、[治理框架与标准](11-AI治理与伦理/245-治理框架与标准.md)：伦理岗和产品岗会问「你们怎么做对齐」「出问题谁负责」「有没有偏见评估」。
