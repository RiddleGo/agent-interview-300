# 第 165 题：数据库与数据源集成

## 题目

Agent/RAG 如何与数据库及各类数据源集成？

---

## 一、集成形态

- **只读查询**：Agent 通过 **Tool** 将自然语言转 **SQL/查询**（Text-to-SQL），在**只读副本或只读账号**上执行，结果返回 Agent 做总结或再推理；避免写操作误伤数据。  
- **RAG 数据源**：将**数据库表/视图**通过 ETL 或**同步**到文档/向量库，供 RAG 检索；或对**查询结果**做摘要后注入 prompt；需增量更新与一致性策略。  
- **写操作**：若需**更新数据**（如工单状态、审批结果），通过**固定 API 或存储过程**封装，Agent 只调 API 不直接写库；API 内做校验、事务与审计。  
- **多源**：同时接 **MySQL、PostgreSQL、Elasticsearch、API** 等；按**数据源类型**路由（如结构化用 SQL、全文用 ES）；权限与配额按源控制。

---

## 二、安全与性能

- **权限**：数据库账号**最小权限**（只读或仅允许访问指定表）；按**租户/用户**过滤（如 RLS、应用层带 tenant_id）。  
- **注入**：**禁止**拼接用户或 Agent 原始文本进 SQL；使用**参数化查询**或 **ORM**；Text-to-SQL 输出需**校验或白名单**（表/列/函数）后再执行。  
- **性能**：**慢查询**限制与超时；大结果**分页或截断**；只读走**从库**或缓存，减轻主库压力。  
- **审计**：**查询与写操作**记日志（谁、何时、库、语句摘要、影响行数）；敏感库可全量审计。

---

## 三、小结与面试要点

**小结**：集成形态包括只读 Text-to-SQL、RAG 同步、写操作封装与多源路由；安全上最小权限、防注入、审计；性能上超时、分页与读写分离。

**面试要点**：能区分只读与写、RAG 同步与 Text-to-SQL；防 SQL 注入与 Text-to-SQL 校验要强调；审计要提到。

---

## 记忆要点

1. 形态：只读 Text-to-SQL、RAG 同步、写操作封装、多源。  
2. 安全：最小权限、参数化/白名单防注入、审计。  
3. 性能：超时、分页、读写分离。
