# 第 176 题：服务网格与 API 网关

## 题目

Agent 应用如何借助服务网格、API 网关做集成与治理？

---

## 一、API 网关

- **职责**：**统一入口**、**认证鉴权**、**限流**、**路由**、**协议转换**（如 HTTP→gRPC）、**日志与监控**。  
- **与 Agent**：所有**对外 Agent/Chat API** 经网关；网关做 **JWT 校验、API Key、限流**，并**注入 user_id/tenant_id** 后转发；**SSL 终结**与**跨域**在网关处理。  
- **常见**：Kong、APISIX、AWS API Gateway、阿里云 API 网关；支持**插件**（认证、限流、日志）与**动态路由**。  
- **多版本**：网关**按 path/Header** 路由到**不同后端版本**（蓝绿、金丝雀）；与灰度发布结合。

---

## 二、服务网格（Service Mesh）

- **职责**：**服务间**通信的**可观测、安全、弹性**（重试、超时、熔断）；通过 **Sidecar** 透明注入，业务代码少改。  
- **与 Agent**：Agent 服务调 **LLM、向量库、下游 API** 时，由 **Sidecar** 做**负载均衡、重试、熔断、mTLS、Trace**；统一策略与观测。  
- **常见**：Istio、Linkerd、Consul Connect；**流量管理**（超时、重试、分流）与**可观测**（指标、Trace）强。  
- **适用**：**多服务、多语言**的 Agent 架构；希望**策略集中**、**业务无侵入**时选用。

---

## 三、选型与组合

- **仅对外 API**：用 **API 网关** 即可；认证、限流、路由、日志集中。  
- **对内多服务**：加 **服务网格** 做服务间治理与可观测；网关负责“南北向”，网格负责“东西向”。  
- **轻量**：**网关 + 各服务内 SDK**（重试、熔断、Trace）也可替代部分网格能力；运维更简单、控制更显式。

---

## 四、小结与面试要点

**小结**：API 网关负责统一入口、认证限流与路由；服务网格负责服务间可观测、安全与弹性；Agent 场景常用网关对外、可选网格对内；可按规模与运维偏好选型。

**面试要点**：能区分网关与网格的职责（南北 vs 东西）；能举网关的认证、限流与路由；网格的 Sidecar 与熔断要提到。

---

## 记忆要点

1. 网关：统一入口、认证鉴权、限流、路由、日志；南北向。  
2. 网格：服务间、Sidecar、可观测、重试熔断、mTLS；东西向。  
3. 组合：网关对外 + 网格对内；或网关 + SDK 轻量方案。
