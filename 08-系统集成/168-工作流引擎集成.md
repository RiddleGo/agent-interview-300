# 第 168 题：工作流引擎集成

## 题目

Agent 如何与 BPM、工作流引擎集成？

---

## 一、集成场景

- **Agent 触发流程**：用户通过对话**发起**审批、工单、请假等；Agent 解析意图后**调用工作流 API** 创建流程实例，并返回流程单号或链接。  
- **流程中调用 Agent**：流程某节点为“**人工+AI**”或“**AI 处理**”；工作流引擎**调用 Agent API**（如分类、摘要、推荐），再根据返回结果**分支或回写**。  
- **Agent 查询状态**：用户问“我的请假到哪步了”；Agent 通过 **Tool** 调工作流 **查询 API**，拿到当前节点与历史，再生成自然语言回复。  
- **事件联动**：流程**状态变更**（如审批通过）发**事件**，Agent 或通知服务订阅后做后续动作（如发消息、更新知识库）。

---

## 二、集成方式

- **API**：工作流引擎暴露 **REST API**（创建实例、查询、审批、取消）；Agent 通过 **Tool** 或内部服务封装调用；需**认证、幂等与错误码**约定。  
- **回调与 Webhook**：流程到达某节点时，引擎**回调** Agent 或业务服务；Agent 处理完再**回调引擎**推进流程；需重试与超时策略。  
- **统一模型**：若工作流与 Agent 同厂（如某低代码平台），可用**内置节点**“调用 Agent”或“自然语言分支”；减少自建胶水。  
- **数据格式**：**流程上下文**（变量、表单）与 Agent 的**输入输出**约定 **schema**；避免字段歧义与类型错误。

---

## 三、注意点

- **权限**：Agent 调工作流 API 时带**用户身份**或**服务身份**；引擎按**权限**校验是否可创建/审批该流程。  
- **一致性**：**审批、回写**等写操作需**幂等**或**唯一业务键**；避免重复提交。  
- **可观测**：**流程实例 id、节点、Agent 调用**记入 trace；便于排查“流程卡在哪、Agent 返回了什么”。

---

## 四、小结与面试要点

**小结**：Agent 与工作流可“Agent 触发流程、流程中调 Agent、Agent 查状态、事件联动”；集成通过 API、回调与统一模型；需权限、幂等与可观测。

**面试要点**：能举 2～3 个场景；API 与回调两种方式要提到；权限与幂等要强调。

---

## 记忆要点

1. 场景：Agent 触发流程、流程调 Agent、Agent 查状态、事件联动。  
2. 方式：工作流 API、回调/Webhook、统一平台节点。  
3. 注意：权限、幂等、可观测。
