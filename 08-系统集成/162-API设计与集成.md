# 第 162 题：API 设计与集成

## 题目

Agent 应用与外部系统通过 API 集成时，设计要点有哪些？

---

## 一、协议与格式

- **REST/OpenAI 兼容**：对外提供 **HTTP REST** 或 **OpenAI 兼容** 的 Chat/Completions、Embeddings；便于生态与客户端复用；版本通过路径或 Header 区分（如 `/v1/`）。  
- **请求与响应**：**JSON** 为主；统一结构（如 `{ data, error, request_id }`）；错误时带 **code、message、details**；**request_id** 贯穿链路便于排查。  
- **流式**：支持 **stream=true** 时用 **SSE** 或 chunked；约定 event 格式与结束标记；超时与背压需明确。

---

## 二、认证与鉴权

- **认证**：**API Key**（Header）、**OAuth 2.0**、**mTLS** 等；多租户时 Key 或 Token 带 **tenant_id** 或由后端解析。  
- **鉴权**：请求带身份后，按 **RBAC/ABAC** 校验是否可访问该资源；未授权返回 403。  
- **限流**：**Rate Limit**（按 Key/租户/IP）；返回 429 与 **Retry-After**；避免滥用与过载。

---

## 三、与下游系统对接

- **调用下游**：Agent 通过 **Tool** 调企业 API 时，需**统一封装**（重试、超时、熔断、日志）；**敏感参数**（如 Key）由服务端注入，不从前端下传。  
- **幂等**：写操作支持 **idempotency_key**；下游不支持时在网关或 Agent 层做**去重**。  
- **错误与降级**：下游不可用时的**错误码映射**与**降级策略**（如返回缓存、默认提示、或明确“服务暂不可用”）。

---

## 四、小结与面试要点

**小结**：API 集成需统一协议与格式、认证鉴权与限流、request_id 与错误结构；调用下游时统一封装、幂等与降级；安全与可观测贯穿。

**面试要点**：能说清 OpenAI 兼容与 REST 的取舍；认证与限流要提到；下游调用的重试、幂等与降级要举一二。

---

## 记忆要点

1. 协议：REST/OpenAI 兼容、JSON、request_id、流式 SSE。  
2. 安全：认证（Key/OAuth）、鉴权、限流与 429。  
3. 下游：统一封装、幂等、错误与降级。
