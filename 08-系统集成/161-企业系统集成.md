# 第 161 题：企业系统集成概述

## 题目

Agent/LLM 应用与企业现有系统集成时，需要考虑哪些方面？

---

## 一、集成的目标

将 **Agent/LLM 能力** 嵌入企业现有 IT 环境：与 **CRM、ERP、工单、知识库、数据库、API** 等打通，实现“用自然语言查数据、办流程、调服务”，同时满足 **安全、合规、审计与运维** 要求。

---

## 二、主要维度

- **身份与权限**：与 **企业 SSO/LDAP** 对接，统一认证；权限与 **RBAC/ABAC** 对齐，Agent 调用下游系统时带**身份与权限上下文**，避免越权。  
- **数据与 API**：通过 **API、数据库只读/读写、消息队列** 等访问业务数据；需约定**数据格式、幂等与错误码**；敏感数据脱敏或按权限过滤。  
- **工作流与流程**：与 **BPM、工单、审批流** 集成；Agent 可触发流程、查询状态、或作为某环节的“智能处理”；需明确**触发条件、输入输出与回写**。  
- **可观测与审计**：**调用链、日志、审计记录** 与企业日志与监控平台打通；满足合规与溯源。  
- **部署与网络**：**内网/专线** 部署或访问；与 **DMZ、防火墙、API 网关** 策略一致；多环境（开发/测试/生产）隔离。

---

## 三、常见模式

- **Agent 作为调用方**：Agent 通过 **Tool** 调用企业 API 或数据库，结果用于推理与生成；需鉴权、限流与错误处理。  
- **Agent 作为服务**：企业应用**调用 Agent API**（如对话、任务执行）；需统一协议（如 OpenAI 兼容）、认证与多租户。  
- **事件驱动**：企业系统**发事件**（如工单创建、审批完成），触发 Agent 处理或通知；通过消息队列或 Webhook 对接。  
- **数据同步**：将企业数据**同步到向量库/知识库**供 RAG；需增量更新、一致性及权限过滤。

---

## 四、小结与面试要点

**小结**：企业集成要兼顾身份与权限、数据与 API、工作流、可观测与审计、部署与网络；常见模式有 Agent 调企业系统、企业调 Agent、事件驱动与数据同步；安全与合规贯穿全流程。

**面试要点**：能说出 3～4 个维度（身份、数据/API、工作流、可观测）；能举 1～2 种集成模式（如 Tool 调 API、事件驱动）；权限与审计必提。

---

## 记忆要点

1. 维度：身份与权限、数据与 API、工作流、可观测与审计、部署与网络。  
2. 模式：Agent 调企业、企业调 Agent、事件驱动、数据同步。  
3. 贯穿：安全、合规、审计。
