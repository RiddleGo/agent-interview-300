# 第 163 题：多模态输入输出集成

## 题目

Agent 应用如何集成多模态（图像、语音、表格）的输入与输出？

---

## 一、输入集成

- **图像**：请求中带 **image URL 或 Base64**；按模型约定格式（如 GPT-4V、Gemini 的 multipart 或 JSON 内嵌）组装；大图可**压缩或分块**以控制 token 与成本。  
- **语音**：**ASR** 将语音转为文本再进 LLM；或使用**多模态模型**直接收音频；需约定编码格式（如 PCM、opus）、采样率与长度限制。  
- **表格**：以 **Markdown 表、CSV 或结构化 JSON** 注入 prompt；或先**描述/摘要**再注入；大表需截断或分层（摘要 + 按需展开）。  
- **文档**：PDF/Office 经**解析与分块**后进 RAG 或直接注入；多模态模型可对**文档截图**做视觉理解。

---

## 二、输出集成

- **文本为主**：多数场景仍以**文本**回复；可带**结构化片段**（如 JSON、Markdown 表）供前端渲染。  
- **图像生成**：若需**生图**，调用 **DALL·E、Stable Diffusion** 等 API，将生成结果以 URL 或 Base64 返回；需与合规与内容策略一致。  
- **语音合成**：**TTS** 将回复转为语音；与 ASR 组合实现“语音对话”；需选音色、格式与延迟权衡。  
- **多模态引用**：回复中**引用**来源（如“根据图 2”“见下表”）；前端按类型高亮或展示对应资源。

---

## 三、管道与一致性

- **端到端管道**：输入 **多模态 → 统一表示或分路处理 → LLM/Agent → 多模态输出**；各环节**格式约定**（如 base64 编码、MIME 类型）与**错误处理**（如图片过大、格式不支持）需统一。  
- **与现有系统**：企业已有**媒体存储、CDN、TTS/ASR 服务**时，Agent 通过 **Tool 或内部 API** 调用；权限与配额与企业策略一致。

---

## 四、小结与面试要点

**小结**：多模态输入包括图像、语音、表格与文档的格式约定与预处理；输出包括文本、生图、TTS 与引用；需端到端管道一致及与企业媒体服务对接。

**面试要点**：能举 2～3 种模态的输入与输出方式；格式约定与 token/成本控制要提到；与企业 ASR/TTS/存储的对接可举一二。

---

## 记忆要点

1. 输入：图像 URL/Base64、语音 ASR、表格 Markdown/JSON、文档解析。  
2. 输出：文本、生图、TTS、多模态引用。  
3. 管道：格式约定、错误处理、与企业服务对接。
