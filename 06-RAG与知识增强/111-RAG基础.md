# 第 111 题：RAG 基础与流程

## 题目

简述 RAG（Retrieval-Augmented Generation）的基本流程与核心组件。

---

## 一、RAG 是什么

**RAG** = **检索（Retrieval）** + **增强（Augmented）** + **生成（Generation）**：在生成回答前，先从外部知识库中**检索**与当前问题相关的文档片段，把这些片段作为**上下文**拼入 prompt，再让大模型基于“问题 + 检索到的内容”**生成**答案。目的是**弥补模型知识截止与事实性不足**，减少幻觉、并支持私有/领域知识。

---

## 二、基本流程

1. **索引阶段（离线）**  
   - **文档摄入**：将原始文档（PDF、网页、数据库等）接入流水线。  
   - **解析与清洗**：提取文本、表格、结构；去噪、归一化。  
   - **分块（Chunking）**：把长文档切成适当大小的片段（chunk），便于检索与放入 context。  
   - **向量化**：对每个 chunk 做 **embedding**，写入**向量库**；可选同时建关键词/稀疏索引。

2. **检索阶段（在线）**  
   - **查询**：用户提出问题（query）。  
   - **查询处理**：可选对 query 做改写、扩展、或生成假设性文档（HyDE）等。  
   - **检索**：用 query（或其向量）在向量库/关键词索引中检索，得到 **top-k** 个相关 chunk。  
   - **重排序（可选）**：用 Reranker 模型对 top-k 精排，保留最相关的若干条。

3. **生成阶段（在线）**  
   - **上下文组装**：将检索到的 chunk、系统指令、用户 query 按约定格式拼成 **prompt**。  
   - **生成**：调用 LLM 生成回答；可要求模型**引用来源**（如 chunk id 或段落）。  
   - **后处理**：可选校验引用、过滤敏感信息、格式化输出。

---

## 三、核心组件小结

| 组件 | 作用 |
|------|------|
| 文档解析与分块 | 把原始文档变成可检索的 chunk |
| 向量化与索引 | 建立语义检索能力（稠密向量）；可辅以稀疏检索 |
| 检索与重排序 | 召回相关 chunk 并精排 |
| 上下文组装与生成 | 将 chunk + query 送入 LLM 生成答案 |

---

## 小结与面试要点

**小结**：RAG = 离线建索引（解析→分块→向量化）+ 在线检索（query 处理→检索→重排）+ 上下文组装与生成；核心是“用检索到的知识增强 prompt，让生成有据可依”。

**面试要点**：能画出“索引→检索→生成”三阶段；说清分块、向量检索、重排序、上下文组装各自作用；RAG 解决的是知识截止与幻觉问题。

---

## 记忆要点

1. RAG = 检索 + 增强（上下文）+ 生成；解决知识截止与幻觉。  
2. 流程：索引（解析、分块、向量化）→ 检索（query 处理、召回、重排）→ 生成（组装 prompt、LLM、引用）。  
3. 核心组件：分块、向量索引、检索与重排、prompt 组装。
