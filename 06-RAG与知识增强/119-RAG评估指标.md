# 第 119 题：RAG 评估指标

## 题目

RAG 系统有哪些常用评估指标？如何评估检索与生成质量？

---

## 一、检索阶段指标

- **召回率（Recall）**：在**标准相关文档集合**已知的前提下，检索结果中有多少比例的相关文档被召回；Recall@K = 相关文档中被召回的个数 / 总相关文档数。  
- **准确率 / 精确率（Precision）**：检索到的 top-K 中有多少是相关文档；Precision@K = top-K 中相关文档数 / K。  
- **MRR（Mean Reciprocal Rank）**：第一个相关文档的排名倒数；对多 query 取平均，反映“相关结果是否靠前”。  
- **NDCG（Normalized DCG）**：考虑排序位置的折损，对多级相关性（如 0/1/2）友好；常用 NDCG@K。  
- **Hit Rate**：top-K 中是否至少有一个相关文档（0/1）；简单直观。

检索评估需**标注数据**：每个 query 对应的相关 doc/chunk 集合（或相关性等级）。

---

## 二、生成阶段与端到端指标

- **忠实度（Faithfulness）**：生成内容是否**忠于**检索到的上下文，有无凭空捏造；可用 NLI 模型或 LLM 判断“陈述是否被上下文支持”。  
- **相关性（Relevance）**：生成答案是否**切题**、是否回答用户问题；可用 LLM 或人工打分。  
- **引用准确率**：若要求模型引用来源，检查引用是否对应真实检索 chunk、是否张冠李戴。  
- **人工评估**：从流畅度、有用性、安全性等维度打分；可做 A/B 与回归基准。

---

## 三、综合与业务指标

- **端到端准确率**：在**有标准答案**的数据集上，比较模型输出与标准答案（精确匹配、F1、或 LLM/人工判断是否等价）。  
- **延迟与成本**：平均/ P99 延迟、单次请求 token 数或 API 成本；在满足质量前提下优化。  
- **Bad Case 分析**：检索漏召、错排、生成幻觉、引用错误等分类统计，指导迭代。

---

## 四、小结与面试要点

**小结**：检索看 Recall/Precision/MRR/NDCG；生成看忠实度、相关性、引用；端到端看准确率与业务指标；需标注数据与 Bad Case 分析。

**面试要点**：能区分检索指标与生成指标；忠实度、引用准确率是 RAG 特有重点；能提到标注与人工评估的作用。

---

## 记忆要点

1. 检索：Recall@K、Precision@K、MRR、NDCG。  
2. 生成：忠实度、相关性、引用准确率。  
3. 端到端：准确率、延迟与成本、Bad Case 分析。
