# 第 112 题：RAG 检索策略——稠密检索、稀疏检索与混合检索

## 题目

解释 RAG 中稠密检索、稀疏检索与混合检索的区别与适用场景。

---

## 一、稠密检索（Dense Retrieval）

**做法**：用**向量模型**（如 BGE、E5、OpenAI Embedding）将 query 和文档 chunk 编码成**稠密向量**，在向量库中做**相似度检索**（如余弦、内积），取 top-k。

**特点**：基于**语义**，能匹配“表述不同但意思相近”的 query 与文档；对同义词、改写、泛化较好。  
**缺点**：对专有名词、数字、精确匹配不如稀疏；依赖 embedding 质量与数据分布。

**适用**：开放域问答、语义相似检索、多语言；RAG 中最常用的主召回方式。

---

## 二、稀疏检索（Sparse Retrieval）

**做法**：用**词袋/词频**表示 query 与文档，基于**词重叠**或 **BM25** 等稀疏表示检索；不经过向量模型，直接在倒排索引上计算分数。

**特点**：**精确词匹配**强，适合专有名词、型号、代码、数字；可解释（哪些词命中）；延迟与资源开销相对可控。  
**缺点**：无法捕捉语义同义与改写；词不重叠则难以召回。

**适用**：关键词明确、精确匹配重要（如产品名、条款号）；常与稠密检索组成**混合检索**。

---

## 三、混合检索（Hybrid）

**做法**：**同时**做稠密检索与稀疏检索（如 BM25），得到两份候选列表，再按规则**融合**：  
- **线性融合**：$s = \alpha \cdot \text{norm}(s_{\text{dense}}) + (1-\alpha) \cdot \text{norm}(s_{\text{sparse}})$，按融合分排序。  
- **RRF（Reciprocal Rank Fusion）**：按排名位置加权，不依赖分数尺度，鲁棒。  
- 也可用**小模型**学习融合权重。

**特点**：兼顾语义与精确匹配，召回更全面，常能提升 RAG 效果。  
**代价**：双路检索与融合计算，延迟与成本略增。

**适用**：既有语义问法又有专有名词/精确词的业务；对效果要求高、可接受略高延迟时。

---

## 四、小结与面试要点

**小结**：稠密 = 向量语义检索；稀疏 = 词匹配/BM25；混合 = 双路召回 + 分数或排名融合；按业务在语义、精确与成本间权衡。

**面试要点**：能区分稠密与稀疏的优缺点；混合用线性融合或 RRF；RAG 中混合检索常优于单路。

---

## 记忆要点

1. 稠密：向量语义，适合语义相似；稀疏：词匹配/BM25，适合精确词。  
2. 混合：双路召回 + 线性融合或 RRF，兼顾语义与精确。  
3. RAG 中混合常提升效果，代价是延迟与资源略增。
