# 第 127 题：RAG 安全与权限控制

## 题目

RAG 系统如何设计安全与权限控制？多租户与敏感文档如何隔离？

---

## 一、要保护的对象

- **数据**：文档与 chunk 的**访问权限**——谁可以检索到哪些文档、哪些字段可返回。  
- **查询**：防止**越权查询**（用 A 身份查到 B 的数据）、**注入与滥用**（恶意 query 探测或爬取）。  
- **输出**：生成结果中**不泄露**未授权内容；引用与溯源不暴露内部结构。

---

## 二、权限与隔离

- **多租户**：每个文档/chunk 带 **tenant_id**（或 org_id、user_id）；检索时**强制带租户过滤**（如 filter tenant_id=当前用户所属租户），保证仅能命中本租户数据；存储与索引按租户分区或加过滤条件。  
- **行级/文档级**：在 tenant 之上可细化为**用户级、项目级、角色级**；文档带 ACL 或标签，检索前解析当前用户权限，只查有权限的集合（如“可读文档 id 列表”或“可读标签”）。  
- **字段级**：敏感字段（如手机号、身份证）**脱敏存储或脱敏返回**；生成阶段可只注入脱敏后的 chunk，或后处理对输出做脱敏。

---

## 三、安全与防护

- **身份与鉴权**：每次请求带**身份**（用户/API Key）；检索与生成前**鉴权**，未授权拒绝。  
- **查询注入**：对用户 query 做**校验与过滤**，防止在 query 中拼接恶意过滤条件（如“忽略 tenant_id”）或 prompt 注入；检索接口不直接暴露“裸 filter”，由服务端按身份注入 filter。  
- **审计**：记录**谁、何时、查了哪些 doc/chunk、返回条数**等，便于溯源与合规；敏感操作可额外告警。

---

## 四、小结与面试要点

**小结**：RAG 安全与权限包括多租户与行级隔离、字段脱敏、身份鉴权、查询注入防护与审计；检索必须带权限过滤，由服务端强制注入不可信任客户端。

**面试要点**：能说清“检索时强制 tenant/ACL 过滤”；多租户与行级权限要提到；查询注入与审计要能举一二。

---

## 记忆要点

1. 隔离：多租户 + 行级/文档级权限；检索强制带 filter。  
2. 安全：身份鉴权、查询注入防护、字段脱敏。  
3. 审计：谁查了啥、何时；敏感操作告警。
