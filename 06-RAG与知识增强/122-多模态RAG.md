# 第 122 题：多模态 RAG

## 题目

多模态 RAG 如何设计？支持图像、表格时的检索与生成有何不同？

---

## 一、多模态 RAG 的含义

**多模态 RAG**：检索与生成不仅针对**文本**，还针对**图像、表格、音频**等；用户 query 可能是文本或图像，知识库中除文本 chunk 外还有图片、图表、表格等，需要**统一或联合检索**，并将多模态上下文注入生成阶段，使模型能“看到”图与表并基于其回答。

---

## 二、检索设计

- **统一向量空间**：用**多模态 embedding 模型**（如 CLIP、ImageBind、多模态 BERT）将文本与图像（及可选表格结构）编码到**同一向量空间**，用同一套向量索引做检索；文本 query 可检索到相关图片，图像 query 可检索到相关文本或图。  
- **分模态检索后融合**：文本用文本 embedding 检索文本 chunk，图像用图像 embedding 检索图像或图文对；两路结果按分数或 RRF 融合，再送入生成。  
- **表格**：表格可转为**结构化描述**（如“列 A、B、C，行 1…”）或单元格级文本做向量检索；或按行列切片成“子表”再向量化；复杂表可结合图或关系型查询。  
- **OCR 与描述**：对图中文字做 **OCR**，对图表做**描述生成**（caption），将描述与原文一起建索引，便于文本 query 命中图像内容。

---

## 三、生成阶段

- **多模态上下文**：将检索到的**文本 chunk + 图像（或图像 URL/Base64）**按模型约定格式拼入 prompt（如 GPT-4V、Gemini 的图文交错格式）；模型同时读文本与图生成答案。  
- **表格**：以 Markdown 表或结构化文本形式注入，或截取子表+说明注入；要求模型“根据下表回答”。  
- **引用**：对多模态结果也需**引用来源**（如“根据图 2”“根据表格 1”），便于溯源。

---

## 四、小结与面试要点

**小结**：多模态 RAG 支持图文表等多种形态；检索可用统一多模态向量或分模态检索再融合；表格可描述化或切片检索；生成时将多模态上下文按模型格式注入并支持引用。

**面试要点**：能说清“统一向量空间”与“分模态融合”两种思路；表格与图像的处理方式要能举例；生成需按多模态模型格式组装 context。

---

## 记忆要点

1. 检索：统一多模态向量 或 分模态检索+融合；表格描述化/切片。  
2. 生成：文本+图像（及表）按模型格式注入；支持多模态引用。  
3. OCR/caption 可将图像内容纳入文本索引。
