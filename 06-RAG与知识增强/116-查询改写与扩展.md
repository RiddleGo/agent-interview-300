# 第 116 题：RAG 查询改写与扩展

## 题目

RAG 中如何进行查询改写（Query Rewriting）与查询扩展（Query Expansion）？各有什么作用？

---

## 一、查询改写（Query Rewriting）

**含义**：在**不改变用户意图**的前提下，将用户 query 改写成更利于检索的形式。

**常见做法**：
- **纠错与归一化**：拼写纠错、繁简统一、同义词替换、去掉口语化表达，使 query 更贴近文档用语。  
- **意图明确化**：多轮场景下把**指代与省略**补全（如“它多少钱”→“产品 X 的价格”）；或把模糊 query 转成明确问句。  
- **多问句拆分**：一个 query 含多个子问题时，拆成多个子 query 分别检索再合并结果。  
- **LLM 改写**：用 LLM 根据指令将 query 改写成“更适合检索的版本”（如关键词突出、结构清晰），可结合 few-shot。

**作用**：提高检索**召回率**与**准确率**，尤其当用户表述与文档表述不一致时。

---

## 二、查询扩展（Query Expansion）

**含义**：在**保留原 query** 的基础上**增加**相关词、同义表述或子 query，用**扩展后的多个 query** 去检索，再**合并/去重**结果，以召回更多相关文档。

**常见做法**：
- **同义词与相关词**：用词表、知识图谱或小模型生成同义词/相关词，拼成多个 query 或加权词表，扩大命中面。  
- **多查询生成（Multi-Query）**：用 LLM 根据原 query 生成若干**不同表述**的 query，分别检索后对结果做融合（如 RRF），提高召回多样性。  
- **假设性文档嵌入（HyDE）**：用 LLM 根据 query **生成一段假设性答案**，用该段落的 embedding 去检索（或与 query embedding 融合）；用“假设答案”与真实文档更接近，常能提升稠密检索效果。

**作用**：缓解**表述鸿沟**与**单一 query 召回不全**的问题，提高召回率与覆盖度。

---

## 三、小结与面试要点

**小结**：改写 = 同一意图下改表述，利于检索；扩展 = 增加同义/多 query/HyDE，扩大召回。二者可组合使用（先改写再扩展），需权衡延迟与成本。

**面试要点**：能区分改写与扩展；改写侧重“更适合检索的表述”，扩展侧重“多路 query 或 HyDE 提高召回”；HyDE 要能简要说明思路。

---

## 记忆要点

1. 改写：纠错、归一化、意图明确化、多问句拆分、LLM 改写。  
2. 扩展：同义词、多 query 生成、HyDE（假设性答案检索）。  
3. 目的：改写提升单次检索质量，扩展提升召回覆盖；可组合使用。
