# 第 117 题：GraphRAG 与知识图谱 RAG

## 题目

什么是 GraphRAG？与基于向量检索的 RAG 有何区别？知识图谱在 RAG 中如何应用？

---

## 一、GraphRAG 的含义

**GraphRAG** 泛指**利用图结构**做检索增强的 RAG：不仅用“文档 chunk 的向量”检索，还利用**实体-关系-实体**组成的图做**多跳推理、社区摘要、全局查询**等，再与生成结合。微软等提出的 GraphRAG 中，会从文档中抽取实体与关系建图，对**社区（子图）**做摘要，查询时既检索局部 chunk，也检索相关社区摘要或子图，适合**需要跨段落推理、全局性问答**的场景。

---

## 二、与向量 RAG 的区别

| 维度 | 向量 RAG | GraphRAG / 图谱 RAG |
|------|----------|----------------------|
| **检索依据** | chunk 与 query 的语义相似度 | 实体、关系、多跳路径、社区 |
| **优势** | 语义匹配、实现简单、易扩展 | 关系推理、多跳、全局与结构化 |
| **适用** | 单段/局部事实、开放域 | 多实体关系、跨文档推理、结构化知识 |
| **成本** | 主要是向量与检索 | 建图、社区摘要、图查询与维护 |

向量 RAG 适合“找相关段落”；GraphRAG 适合“谁和谁什么关系、从 A 经几步到 B、某主题的全局概览”等。

---

## 三、知识图谱在 RAG 中的应用

- **实体与关系抽取**：从文档中抽取实体（人、组织、概念）与关系，写入图数据库（如 Neo4j）；检索时可用“与 query 实体相关的子图”作为上下文。  
- **多跳检索**：query 涉及多跳关系（如“A 的供应商的合规状态”）时，在图上前向/反向遍历，取子图或路径描述再交给 LLM 生成。  
- **与向量结合**：**混合架构**——向量检索召回相关 chunk，同时用图中实体与关系做扩展或过滤；或先用图确定“相关实体/社区”，再在这些实体涉及的 chunk 上做向量检索，缩小范围并提升相关性。  
- **社区摘要**：对图上的**社区（紧密相连的实体团）**用 LLM 做摘要，存储为“社区级知识”；全局性、概述类 query 可优先查社区摘要，再按需下钻到 chunk。

---

## 四、小结与面试要点

**小结**：GraphRAG 利用图结构做检索与推理，适合多跳与全局性问答；与向量 RAG 互补，可混合使用；知识图谱用于实体关系、多跳检索、社区摘要与向量联合。

**面试要点**：能说清 GraphRAG 与向量 RAG 的差异；图适合关系与多跳，向量适合语义与局部；实际常采用“图+向量”混合架构。

---

## 记忆要点

1. GraphRAG = 图结构 + 检索 + 生成；适合多跳与全局。  
2. 与向量 RAG：图重关系与推理，向量重语义与局部；可混合。  
3. 图谱应用：实体关系、多跳检索、社区摘要、与向量联合。
