# 第 28 题：规划（Planning）——单步规划 vs 多步规划

## 题目

什么是规划（Planning）？对比单步规划和多步规划。

---

## 一、什么是规划？

**规划（Planning）** 指 Agent 在执行前或执行过程中，对**如何达成目标**进行分解与排序：要完成哪些子目标、先做什么后做什么、依赖关系如何、失败时如何回退或重试。规划使 Agent 从“走一步看一步”变为“心中有图、按图执行”，尤其适合多步、多工具、有依赖的复杂任务。

在 LLM Agent 中，规划通常表现为：模型生成**步骤序列**或**任务树**（如“先查 A，再根据 A 查 B，最后综合写报告”），再由执行层按序或按依赖调用工具并推进。

---

## 二、单步规划（Single-step / Reactive）

**含义**：不显式生成多步计划，每轮只决定**当前一步**做什么（下一个工具调用或下一段推理），根据执行结果再决定再下一步；即“反应式”、无全局计划。

**特点**：

- **优点**：实现简单、对动态变化适应快、不需要一次性想清全流程；适合环境不确定、目标可随时调整的场景。  
- **缺点**：容易局部最优、遗漏必要步骤、顺序不当导致重复或死循环；复杂任务上效率与成功率都不如“先想好再做”。

**典型**：简单 ReAct、只给“当前该干什么”的 prompt，不显式输出多步计划。

---

## 三、多步规划（Multi-step / Plan-and-Execute）

**含义**：先**显式生成**多步计划（子目标序列或任务树），再**按计划逐步执行**；执行中可根据观察修正或重规划（Replanning）。

**特点**：

- **优点**：全局一致、步骤完整、依赖关系清晰；复杂任务上成功率和效率通常更好；便于人类审查与调试。  
- **缺点**：对模型要求高（要能产出合理计划）、计划可能因环境变化失效，需配合“执行-观察-重规划”的闭环。

**典型**：Plan-and-Execute、HuggingGPT 式“先规划任务列表再执行”、Tree-of-Thought 等；可结合 LLM 生成计划 + 传统规划器（如 PDDL）做约束与验证。

---

## 四、对比与选用

| 维度 | 单步规划 | 多步规划 |
|------|----------|----------|
| **计划形式** | 无显式计划，每步现决策 | 显式多步计划或任务树 |
| **适应性** | 对动态、不确定适应好 | 依赖计划质量，需重规划机制 |
| **复杂任务** | 易漏步、顺序乱、效率低 | 更完整、顺序可控、效率高 |
| **实现** | 简单，单轮决策 | 需计划生成、执行、可选重规划 |

**选用**：简单、步骤少、环境多变的任务可用单步；步骤多、有明确依赖、需可解释流程的任务用多步，并配合执行反馈与重规划。

---

## 五、小结与面试要点

**小结**：规划 = 对如何达成目标做分解与排序；单步规划 = 每轮只决定当前一步，反应式；多步规划 = 先显式生成多步计划再执行，可选重规划；复杂任务多用多步并配合执行反馈。

**面试要点**：

- 规划：分解目标、排顺序、定依赖，再执行。  
- 单步：无显式计划，每步现决策；简单、适应性强，复杂任务易漏步。  
- 多步：先出计划再执行；适合复杂任务，需计划质量与重规划。  
- 实践：简单任务单步即可；多步多工具用多步规划 + 执行-观察-重规划。

---

## 记忆要点

1. 规划 = 目标分解与步骤排序，再执行。  
2. 单步规划：每步现决策，无显式多步计划；简单适应性强，复杂易漏步。  
3. 多步规划：先生成多步计划再执行；适合复杂任务，需重规划配合。
