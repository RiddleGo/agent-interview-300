# 第 32 题：工具学习（Tool Learning）与如何评估 Agent 的工具使用能力

## 题目

什么是工具学习（Tool Learning）？如何评估 Agent 的工具使用能力？

---

## 一、什么是工具学习？

**工具学习（Tool Learning）** 指让模型**学会在何时选何种工具、如何填参、如何根据结果再决策**。包括：

- **工具选择**：给定任务与工具列表，正确选出要调用的工具（可能多个、有顺序）。  
- **参数生成**：根据工具 schema 与任务，生成合法、语义正确的参数（如查询词、API 参数）。  
- **结果利用**：根据工具返回（含错误、空结果）决定下一步——继续调用、换工具、或给出最终答案。  
- **多步编排**：在多步任务中正确编排工具调用顺序与依赖。

实现路径包括：**纯 prompt（zero/few-shot）**、**微调**（用 (任务, 工具调用轨迹) 数据）、**RL**（以任务成功为 reward 优化工具选择与调用）、以及 **API / 工具描述增强**（如 Gorilla、APIBench 等提供高质量 API 文档与示例）。

---

## 二、评估维度

- **工具选择准确率**：在标注了“应调用工具”的数据上，模型是否选对工具（或工具序列）。  
- **参数正确率**：参数是否合法（符合 schema）、是否与任务匹配（如查询词是否相关）。  
- **任务完成率 / 成功率**：端到端执行后是否达成目标（如问答答对、API 任务成功）；可区分“选对工具但参数错”“完全选错”等。  
- **效率**：步数、多余调用次数、超时与失败重试次数；在保证成功率下的平均步数或 token 数。  
- **鲁棒性**：对工具失败、超时、返回异常的恢复能力；是否避免重复无效调用、死循环。  
- **泛化**：在未见过的工具或 API 上的表现（依赖描述质量与模型泛化）。

---

## 三、常用基准与指标

- **API 相关**：ToolBench、APIBench、Gorilla 等，评估模型根据文档选择 API、填参、多步调用的能力；指标有准确率、成功率、通过率。  
- **通用工具**：如 ReAct 式基准，给若干工具（搜索、计算器、DB 等），测多步任务完成率与步骤正确率。  
- **指标**：Task Success Rate、Tool Selection Accuracy、Parameter Accuracy、Steps / Efficiency；可加人工评估（结果可用性、安全性）。

面试可答：评估从“选对工具 → 参数正确 → 任务成功”分层看；既有工具/API 专用基准，也看端到端任务成功与效率、鲁棒性、泛化。

---

## 四、小结与面试要点

**小结**：工具学习 = 学会选工具、填参、用结果再决策及多步编排；评估包括工具选择与参数正确率、任务完成率、效率、鲁棒性、泛化，常用 API/工具基准与分层指标。

**面试要点**：

- 工具学习：学会何时选何工具、如何填参、如何用结果再决策及多步编排。  
- 评估：工具选择/参数正确率、任务完成率、步数/效率、鲁棒性、泛化。  
- 基准：ToolBench、APIBench、Gorilla 等；指标如成功率、准确率、步数。

---

## 记忆要点

1. 工具学习：选工具、填参、用结果再决策、多步编排。  
2. 评估：选择/参数正确率、任务成功率、效率、鲁棒性、泛化。  
3. 基准与指标：API/工具基准（ToolBench 等）、成功率与步数等。
