# 第 21 题：什么是 AI Agent？与单纯 LLM 的核心区别

## 题目

什么是 AI Agent？与单纯的 LLM 相比，其核心区别是什么？

---

## 一、什么是 AI Agent？

**AI Agent（智能体）** 是在环境中**自主感知、决策并执行动作**以完成目标的系统。在 LLM 时代，通常指以**大语言模型为“大脑”**，能**调用外部工具**（搜索、计算器、API、数据库等）、**与环境交互**、**多步推理与规划**，并可根据反馈**调整行为**的智能体。

简单说：**Agent = LLM + 工具 + 环境交互 + 循环（感知→决策→执行）**。它不只会“说”，还会“做”——通过工具把语言转化为行动。

---

## 二、与单纯 LLM 的核心区别

| 维度 | 单纯 LLM | AI Agent |
|------|----------|----------|
| **输出** | 文本（下一 token/回复） | 文本 + **动作**（调用工具、执行操作） |
| **闭环** | 单轮或多轮对话，无“执行-反馈” | **感知→决策→执行→观察**循环，有环境反馈 |
| **工具** | 无（或仅被动被系统调用） | **主动选择**并调用工具，根据结果再决策 |
| **目标** | 生成符合上下文的文本 | **完成用户/系统目标**（订票、查数据、写代码等） |
| **规划** | 多为单轮或短链推理 | **多步规划**、分解子目标、处理失败与重试 |

**一句话**：LLM 是“会说的脑子”；Agent 是“会说、会选工具、会根据结果再说的脑子”，具备**行动能力**和**闭环决策**。

---

## 三、为什么这个区别重要？

- **落地场景**：纯 LLM 适合问答、写作、翻译；Agent 适合需要“动手”的任务：查实时信息、操作软件、订票、写代码并执行等。  
- **架构设计**：Agent 需要工具描述、调用协议、结果解析、错误处理、记忆与状态管理，这些在纯 LLM 系统中往往没有。  
- **面试**：考官常通过“Agent 和 ChatGPT 有什么区别”来考察你是否理解“行动闭环”和“工具使用”。

---

## 四、小结与面试要点

**小结**：AI Agent = 以 LLM 为核心、能主动使用工具、与环境交互并依反馈循环决策的系统；与单纯 LLM 的核心区别是**行动能力**和**感知-决策-执行闭环**。

**面试要点**：

- 定义：能感知、决策、执行并在环境中完成目标的系统；常以 LLM 为核心 + 工具 + 闭环。  
- 与 LLM 区别：LLM 只出文本；Agent 出文本+动作，有工具调用、环境反馈、多步规划与目标导向。  
- 一句话：Agent = LLM + 工具 + 闭环（感知→决策→执行→观察）。

---

## 记忆要点

1. Agent = 在环境中自主感知、决策、执行以完成目标的系统。  
2. 与 LLM：LLM 只生成文本；Agent 生成文本+调用工具，有闭环与目标。  
3. 关键能力：工具使用、环境交互、多步规划、根据反馈调整。
