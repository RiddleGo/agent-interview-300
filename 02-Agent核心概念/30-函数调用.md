# 第 30 题：函数调用（Function Calling）与工具使用的关系

## 题目

什么是函数调用（Function Calling）？与工具使用的关系是什么？

---

## 一、什么是函数调用（Function Calling）？

**函数调用** 指模型在生成回复时，**输出结构化的“调用请求”**（如要调用的函数名、参数），由系统解析后真正执行对应函数（如 API、数据库、本地工具），并把执行结果再交给模型或直接返回用户。API 层面通常提供：**工具/函数的 schema**（名称、描述、参数 JSON Schema）和**模型返回的 tool_calls**（id、函数名、参数 JSON），系统据此路由并执行。

OpenAI 的 Function Calling、Google 的 Tool Use、Claude 的 tools 等，都是这一能力的官方封装：模型输出“调谁、传什么参数”，平台或应用层负责执行并回填结果。

---

## 二、与工具使用（Tool Use）的关系

- **概念上**：Function Calling 是**工具使用的一种实现方式**——工具被抽象成“函数”（名称+参数），模型选择并填参，系统执行。所以“工具使用”是能力概念，“函数调用”是接口/协议层面的实现。  
- **范围**：工具使用可以包含非结构化形式（如 ReAct 里模型输出自然语言 `Action: search[query]`，再由解析器解析）；函数调用特指**结构化**的调用约定（JSON schema + 模型返回结构化 tool_calls），便于自动解析、校验与路由。  
- **实践**：现代 Agent 系统常用“函数调用”作为工具使用的标准接口：把每个工具注册为一个 function，模型按 schema 返回 function name + arguments，系统执行并返回 observation，再继续下一轮。  
- **等价说法**：很多场合下“工具使用”与“函数调用”混用，面试时可以说：函数调用是工具使用的结构化、API 友好实现；工具使用是更上层的“Agent 调用外部能力”的概念。

---

## 三、典型流程

1. **注册**：向模型/API 提供 `tools` 列表，每项含 `name`、`description`、`parameters`（JSON Schema）。  
2. **请求**：用户消息 + 可选历史 + tools 描述送入模型。  
3. **模型输出**：模型返回 `tool_calls`（如 `{ name: "get_weather", arguments: { "city": "北京" } }`）或普通文本回复。  
4. **执行**：系统根据 `tool_calls` 调用真实函数，得到结果。  
5. **回填**：将结果以 `tool_results` 形式再发给模型，模型可继续生成或给出最终回答。

---

## 四、小结与面试要点

**小结**：函数调用 = 模型输出结构化的“调谁、传什么参数”，由系统执行并回填结果；与工具使用的关系是：工具使用是能力概念，函数调用是结构化、API 级的实现方式，现代 Agent 常用函数调用实现工具使用。

**面试要点**：

- 函数调用：模型输出结构化调用（函数名+参数），系统执行并回填。  
- 与工具使用：工具使用是“调用外部能力”的概念；函数调用是结构化实现，常作为工具使用的标准接口。  
- 流程：注册 tools schema → 模型返回 tool_calls → 系统执行 → 结果回填。

---

## 记忆要点

1. 函数调用 = 模型输出结构化“函数名+参数”，系统执行并回填。  
2. 与工具使用：工具使用是概念，函数调用是结构化实现方式。  
3. 现代 Agent 常用 Function Calling 作为工具使用的标准接口。
