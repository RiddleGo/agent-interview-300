# 第 34 题：Agent 中的目标分解（Goal Decomposition）策略

## 题目

解释 Agent 中的目标分解（Goal Decomposition）策略。

---

## 一、什么是目标分解？

**目标分解（Goal Decomposition）** 指把**高层目标**拆成若干**子目标**（或子任务），并明确顺序与依赖，使每个子目标可被单独执行或分配给专门模块/Agent，最终通过组合子结果达成总目标。这是规划与多步推理的核心环节：Agent 不能一步完成时，必须先“想清楚要分几步、每步做什么”。

在 LLM Agent 中，分解通常由 **LLM 根据用户目标与可用工具** 生成：输出结构化的子目标列表或任务 DAG，再由执行层按序或按依赖调用工具、汇总结果。

---

## 二、常见策略

- **顺序分解**：按时间或逻辑顺序拆成步骤 1、2、3…，前一步输出可作为后一步输入（如先检索再总结）。  
- **层次分解**：顶层目标 → 若干中层子目标 → 每个中层再拆为更细步骤；形成树或 DAG，便于并行与依赖管理。  
- **依赖标注**：显式标出“子任务 A 依赖 B 的输出”，执行时按拓扑序或动态调度，避免循环与缺失输入。  
- **工具导向**：根据可用工具反推“要完成目标需要哪些能力”，再为每种能力分配子目标（如需要搜索 → 子目标“获取实时信息”）。  
- **LLM 生成**：用 prompt 让模型输出“为完成 X，应依次做：1…2…3…”或结构化 JSON；可 few-shot 给分解示例，提高稳定与合理性。

---

## 三、设计要点

- **可执行**：每个子目标应对应具体动作或工具调用，避免过于抽象无法落地。  
- **无遗漏**：分解后合起来应覆盖总目标，必要时加“综合/汇总”子目标。  
- **可验证**：子目标最好有清晰完成条件，便于判断是否达成、是否需重试。  
- **依赖一致**：顺序与依赖要与工具与数据流一致，避免循环依赖或缺失输入。

面试可答：目标分解是把大目标拆成可执行的子目标并定顺序与依赖；策略包括顺序/层次分解、依赖标注、工具导向、LLM 生成；要保证可执行、无遗漏、可验证与依赖一致。

---

## 四、小结与面试要点

**小结**：目标分解 = 把高层目标拆成子目标并定顺序与依赖；策略有顺序/层次分解、依赖标注、工具导向、LLM 生成；设计上要可执行、无遗漏、可验证、依赖一致。

**面试要点**：

- 定义：把总目标拆成子目标，定顺序与依赖，便于逐步执行与组合。  
- 策略：顺序/层次、依赖标注、工具导向、LLM 生成子目标列表或 DAG。  
- 要点：子目标可执行、覆盖总目标、可验证、依赖一致。

---

## 记忆要点

1. 目标分解 = 总目标 → 子目标 + 顺序与依赖。  
2. 策略：顺序/层次、依赖标注、工具导向、LLM 生成。  
3. 要点：可执行、无遗漏、可验证、依赖一致。
