# 第 39 题：Toolformer、Gorilla、APIBench 在工具学习方面的贡献

## 题目

对比 Toolformer、Gorilla、APIBench 在工具学习方面的贡献。

---

## 一、Toolformer

**贡献**：用**自监督**方式让 LLM 学会**何时调用工具、如何构造调用**。做法：在大量文本中自动插入“工具调用 + 结果”的占位，用真实工具执行得到结果，再让模型学习“在合适位置预测工具调用并利用结果”；从而模型学会在推理中插入 calculator、搜索引擎、翻译等工具调用，且**不依赖大量人工标注**的 (任务, 工具轨迹) 数据。

**特点**：强调“工具调用与文本生成统一建模”、通过自监督从文本与工具执行中学习；为后续“模型内建工具使用”提供范式。

---

## 二、Gorilla

**贡献**：面向 **API 调用** 的工具学习，重点解决“**选对 API**”和“**正确填参**”。提供 Gorilla 模型与 **APIBench** 数据集：APIBench 包含大量 API 文档与 (自然语言请求, 正确 API+参数) 的标注，用于训练/评估模型根据用户意图选择 API 并生成正确参数。Gorilla 在 API 选择与参数生成上表现突出，且能较好**泛化到训练时未见过的 API**（通过文档理解）。

**特点**：API 专用、强调文档驱动与泛化；APIBench 成为 API/工具学习常用基准。

---

## 三、APIBench

**贡献**：作为**基准数据集与评估框架**：包含多类 API（如云服务、知识库）的文档与 (用户查询, 正确 API 调用) 标注，用于评估模型是否选对 API、参数是否正确、以及能否泛化到新 API。常与 Gorilla、ToolBench 等工作一起被引用，推动“工具/API 学习”的可比性与迭代。

**特点**：偏基准与评估；与 Gorilla 等模型结合形成“数据+模型+评估”的完整链条。

---

## 四、对比小结

| 工作 | 侧重 | 主要贡献 |
|------|------|----------|
| **Toolformer** | 通用工具、自监督 | 自监督学习“何时何地调用工具”、统一工具与文本生成 |
| **Gorilla** | API 调用、文档驱动 | 强 API 选择与参数生成、泛化到未见 API；配合 APIBench |
| **APIBench** | 基准与评估 | API 工具学习的数据集与评估标准 |

面试可答：Toolformer 做自监督工具学习、统一工具与生成；Gorilla 做 API 专用、文档驱动与泛化；APIBench 提供 API 工具学习的基准数据与评估。

---

## 五、小结与面试要点

**小结**：Toolformer 用自监督学工具调用与结果利用；Gorilla 专注 API 选择与参数、文档驱动与泛化；APIBench 提供 API 工具学习基准；三者分别代表通用自监督、API 专用与基准评估。

**面试要点**：

- Toolformer：自监督、何时何地调用工具、统一工具与文本。  
- Gorilla：API 选择与参数、文档驱动、泛化到新 API；常配 APIBench。  
- APIBench：API 工具学习的基准数据集与评估。

---

## 记忆要点

1. Toolformer：自监督工具学习，统一工具调用与文本生成。  
2. Gorilla：API 专用，文档驱动，选 API+填参，泛化到新 API。  
3. APIBench：API 工具学习基准与评估。
