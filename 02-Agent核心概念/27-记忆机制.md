# 第 27 题：Agent 中的记忆机制——短期记忆 vs 长期记忆

## 题目

解释 Agent 中的记忆机制：短期记忆 vs 长期记忆，如何实现？

---

## 一、为什么 Agent 需要记忆？

Agent 在**多轮对话、多步任务**中需要记住：当前对话说了什么、之前调用过哪些工具、结果是什么、用户偏好与历史决策等。单靠“当前 prompt 窗口”会受长度限制且无法跨会话持久化，因此需要**短期记忆**（当前会话内的状态）与**长期记忆**（跨会话、可持久的知识与经历）的区分与实现。

---

## 二、短期记忆（Short-term / Working Memory）

**含义**：当前任务或当前会话内的信息——对话轮次、本轮及前几轮的 Thought/Action/Observation、当前子目标、临时变量等。

**实现方式**：

- **滑动窗口**：只保留最近 N 轮或最近 K 个 token 的对话与工具结果，拼入 prompt；实现简单，但较早信息会丢失。  
- **摘要压缩**：对较早的对话或长 Observation 做摘要，保留摘要 + 最近原文，平衡上下文长度与信息量。  
- **结构化状态**：在代码中维护“当前任务栈、已执行步骤、工具结果列表”等，按需选择性注入 prompt，而不是整段原始历史。

短期记忆的目标是让模型在**当前会话内**不遗忘刚发生的事，且不超出上下文限制。

---

## 三、长期记忆（Long-term Memory）

**含义**：跨会话、可持续利用的信息——用户画像、历史偏好、过去任务的关键结论、学到的领域知识或工具使用模式等。

**实现方式**：

- **向量库**：把历史对话、重要结论、用户偏好等写成片段，做 embedding 存入向量库；当前会话用当前 query 检索相关记忆，再拼入 prompt（类似 RAG）。  
- **知识图谱 / 结构化存储**：实体、关系、事件等结构化存储，按需查询注入。  
- **外部数据库**：用户表、行为表、配置表等，通过工具或 API 查询，相当于“记忆外挂”。  
- **MemGPT 等**：显式区分“主上下文”与“记忆层”，用检索或层级管理把长期记忆按需调入工作区。

长期记忆的目标是**跨会话复用**、支持个性化与持续学习，且不把所有历史都塞进单次 prompt。

---

## 四、短期 vs 长期对比与组合

| 维度 | 短期记忆 | 长期记忆 |
|------|----------|----------|
| **时间范围** | 当前会话/当前任务 | 跨会话、持久 |
| **内容** | 对话、工具调用与结果、当前推理链 | 用户画像、历史结论、偏好、知识 |
| **实现** | 滑动窗口、摘要、结构化状态 | 向量库、知识图谱、DB、MemGPT 式层级 |
| **典型用途** | 保证本轮推理连贯、不丢步 | 个性化、持续学习、知识积累 |

实际系统往往**两者结合**：短期用窗口+摘要维持当前会话状态，长期用检索或 DB 按需注入历史与偏好。

---

## 五、小结与面试要点

**小结**：短期记忆 = 当前会话/任务内的状态（对话、工具链、当前目标），可用滑动窗口、摘要、结构化状态实现；长期记忆 = 跨会话持久信息（用户、偏好、知识），可用向量库、图谱、DB 或 MemGPT 式层级实现；两者组合以兼顾连贯性与持久化。

**面试要点**：

- 短期：当前会话内状态，实现如滑动窗口、摘要、结构化状态。  
- 长期：跨会话持久信息，实现如向量检索、知识图谱、DB。  
- 组合：短期保当前连贯，长期做个性化与知识复用；注意上下文长度与检索质量。

---

## 记忆要点

1. 短期记忆：当前会话/任务，对话与工具链；实现有滑动窗口、摘要、结构化状态。  
2. 长期记忆：跨会话持久，用户与知识；实现有向量库、图谱、DB、MemGPT。  
3. 组合使用：短期维持连贯，长期做个性化与知识注入。
