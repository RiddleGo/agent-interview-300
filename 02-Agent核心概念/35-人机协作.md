# 第 35 题：人机协作（Human-in-the-loop）在 Agent 系统中的设计

## 题目

什么是人机协作（Human-in-the-loop）？在 Agent 系统中如何设计？

---

## 一、什么是 Human-in-the-loop？

**Human-in-the-loop（HITL）** 指在自动决策或执行流程中**引入人的参与**：由人在关键节点进行确认、修正、审批或提供反馈，再继续自动化流程。目的是在保持效率的同时，提高**安全性、可控性与合规性**，避免 Agent 误操作、幻觉或越权。

在 Agent 系统中，HITL 常表现为：工具执行前需人审批、敏感操作前二次确认、结果呈现后人可纠正或补充、或人给出偏好/反馈以调整后续行为。

---

## 二、在 Agent 系统中如何设计

- **审批点**：在特定类型的动作前（如发邮件、支付、删数据）插入“等待人工审批”；Agent 输出待执行动作，系统挂起并通知人，人通过/拒绝/修改后再继续。  
- **确认与二次确认**：对高风险操作要求显式确认（如“确认执行 Y？”）；可选“默认拒绝、人点同意才执行”的策略。  
- **结果审核与修正**：Agent 输出（如报告、代码、决策建议）先给人审阅，人可编辑、驳回或通过；通过后再作为最终输出或作为下一步输入。  
- **反馈环**：人可对 Agent 的中间或最终结果打标签（对/错、满意/不满意）或自然语言反馈；系统将反馈写入记忆或用于微调/偏好学习，影响后续行为。  
- **权限与范围**：明确“哪些动作必须经人、哪些可自动”；按角色或敏感度配置不同 HITL 策略（如仅高敏感操作需审批）。  
- **降级与兜底**：当 Agent 不确定、失败或超出范围时，自动转人工处理（如转人工客服、任务挂起待人处理），并保留上下文供人接手。

设计时要权衡**响应速度与安全性**：审批越多越安全但延迟越大；可对操作分级，仅高风险走 HITL。

---

## 三、小结与面试要点

**小结**：HITL = 在关键节点引入人的确认、审批、修正或反馈；在 Agent 中可通过审批点、二次确认、结果审核与修正、反馈环、权限与范围、以及失败时转人工等设计实现，并需权衡延迟与安全。

**面试要点**：

- 定义：关键节点引入人的确认、审批或反馈，保证安全与可控。  
- 设计：审批点、二次确认、结果审核与修正、反馈环、权限分级、失败转人工。  
- 权衡：审批越多越安全但延迟越大；可按操作风险分级设计。

---

## 记忆要点

1. HITL = 关键节点引入人的确认、审批、修正或反馈。  
2. 设计：审批点、二次确认、结果审核、反馈环、权限与转人工。  
3. 权衡：安全 vs 延迟；可按风险分级。
