# 第 36 题：Agent 的自主性（Autonomy）谱系：从简单反射到目标驱动

## 题目

解释 Agent 的自主性（Autonomy）谱系：从简单反射到目标驱动。

---

## 一、自主性谱系是什么？

**自主性谱系** 描述 Agent **“自己决定、自己行动”的程度**：从几乎无自主（纯反射、严格按规则）到高度自主（自定目标、长期规划、主动探索）。常用于区分不同形态的 Agent 与系统设计边界。

---

## 二、从低到高的典型层级

- **反射/反应型**：输入 → 固定规则或简单映射 → 输出；无内部状态与目标，不“想”只“反应”。例：关键词触发固定回复、简单 if-then 规则。  
- **有限状态/脚本型**：按预定义状态或脚本执行，有状态转移但无开放目标；自主性限于在给定流程内选择分支。例：流程固定的客服脚本、简单工作流。  
- **目标驱动但无规划**：有明确单步目标（如“回答这个问题”），每次根据当前输入与目标做一次决策，无显式多步规划。例：单轮 QA、单次工具调用。  
- **目标驱动 + 规划**：有明确目标并能**分解与规划**多步，根据执行反馈调整；仍在一个“给定目标”下自主决定如何达成。例：ReAct、Plan-and-Execute 类 Agent。  
- **目标与动机自定**：Agent 自身具有或可生成**长期目标与动机**，主动选择要做什么、何时探索与利用；高度自主，接近“自主智能体”研究愿景。例：部分强化学习 Agent、研究中的“自主目标设定”系统。

当前工业界 LLM Agent 多处于“目标驱动 + 规划”层：用户或系统给目标，Agent 自主规划与执行；真正的“自定目标”仍以研究为主。

---

## 三、面试中如何答

- 谱系描述的是**自主程度**：从反射、脚本、单步目标、到规划型目标驱动、再到自定目标。  
- 当前主流 Agent：用户给目标，Agent 自主规划与工具使用，属“目标驱动 + 规划”；不是纯反射也不是完全自定目标。  
- 设计时需明确**自主边界**：哪些由人定、哪些由 Agent 定，以平衡能力与安全。

---

## 四、小结与面试要点

**小结**：自主性谱系从低到高大致为：反射型 → 脚本/状态型 → 单步目标型 → 目标+规划型 → 自定目标型；当前 LLM Agent 多为目标驱动+规划；设计需明确自主边界。

**面试要点**：

- 谱系：反射 → 脚本 → 单步目标 → 目标+规划 → 自定目标。  
- 当前 Agent：多为“用户给目标，Agent 自主规划与执行”。  
- 设计：明确自主边界，平衡能力与安全。

---

## 记忆要点

1. 自主性谱系：反射 → 脚本 → 单步目标 → 目标+规划 → 自定目标。  
2. 当前 LLM Agent：目标驱动 + 规划，用户给目标、Agent 自主规划与执行。  
3. 设计需明确自主边界。
