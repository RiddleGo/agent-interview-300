# 第 26 题：多 Agent 系统（Multi-Agent System）及其优势

## 题目

什么是多 Agent 系统（Multi-Agent System）？其优势是什么？

---

## 一、什么是多 Agent 系统？

**多 Agent 系统（Multi-Agent System, MAS）** 指由**多个 Agent** 共同协作完成任务的架构：每个 Agent 具备一定的感知、决策与执行能力（通常以 LLM 为核心），通过**任务分配、通信、协调**完成单 Agent 难以高效完成的复杂目标。

与单 Agent 的对比：单 Agent 在一个大脑内完成推理与工具调用；多 Agent 则把**角色、专长或子任务**拆分到不同 Agent（如“检索员”“写代码的”“审核的”），通过消息传递或共享状态协同。

---

## 二、典型形态

- **角色分工**：不同 Agent 承担不同角色（如 PM、工程师、测试），模拟团队协作。  
- **任务分解**：上层 Agent 或编排器把目标拆成子任务，分配给专门 Agent 执行，再汇总结果。  
- **通信**：Agent 之间通过结构化消息（任务描述、结果、请求帮助）交换信息；可有中心协调者（Centralized）或对等协商（Decentralized）。  
- **代表框架**：AutoGen、CrewAI、MetaGPT 等，都提供多 Agent 编排、角色定义与对话/任务流。

---

## 三、优势

- **能力分工**：不同 Agent 专注不同子领域或工具集，避免单模型“什么都会一点、什么都不精”。  
- **复杂任务分解**：大目标拆成子目标，并行或流水线执行，适合软件开发、调研、多步骤决策等。  
- **鲁棒与可扩展**：单点失败可替换或重试某个 Agent；新增能力可加新 Agent 而非改单一大模型。  
- **可解释与可控**：角色清晰、消息可追踪，便于审计和约束（如“只有审核 Agent 能对外输出”）。  
- **人类协作**：可设计“人类 Agent”参与审批、标注或关键决策，形成人机混合协作。

面试常问：“什么时候用多 Agent 而不是单 Agent？”——任务复杂、需要明确分工或并行、或需要不同专长/权限分离时，多 Agent 更合适。

---

## 四、小结与面试要点

**小结**：多 Agent 系统 = 多个 Agent 通过任务分解、角色分工与通信协同完成任务；优势包括能力分工、复杂任务分解、鲁棒性、可扩展性与可解释性。

**面试要点**：

- 定义：多个 Agent 协作，通过任务分配、通信、角色分工完成复杂目标。  
- 优势：分工、分解复杂任务、鲁棒、可扩展、可解释；适合需要专长分离或并行的大任务。  
- 何时用：任务复杂、需明确分工/并行/权限分离时考虑多 Agent。

---

## 记忆要点

1. 多 Agent 系统 = 多个 Agent 协同，角色分工、任务分解、通信。  
2. 优势：能力分工、复杂任务分解、鲁棒、可扩展、可解释。  
3. 适用：复杂、需分工或并行的任务；代表框架 AutoGen、CrewAI、MetaGPT。
