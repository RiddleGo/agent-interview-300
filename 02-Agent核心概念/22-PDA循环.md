# 第 22 题：Agent 的感知-决策-执行循环（PDA Loop）

## 题目

解释 Agent 的感知-决策-执行循环（Perception-Decision-Action Loop）。

---

## 一、什么是 PDA 循环？

**感知-决策-执行（Perception-Decision-Action, PDA）** 是 Agent 的基本工作循环：

1. **感知（Perception）**：从环境中获取信息——用户输入、工具返回结果、系统状态、历史对话等。  
2. **决策（Decision）**：根据当前感知和内部状态（记忆、目标）决定**下一步做什么**——继续推理、调用某工具、给出最终回答等。  
3. **执行（Action）**：把决策落实——若选工具则发起调用，若结束则输出回复；执行会改变环境或得到新观测。  
4. **回到感知**：执行后的**观察（Observation）**作为新一轮的感知输入，循环直到任务完成或终止。

这是经典控制论与机器人学中的“感知-规划-行动”在 LLM Agent 中的体现：**Agent 不是一次生成到底，而是“看→想→做→再看”的闭环**。

---

## 二、在 LLM Agent 中的具体形式

- **感知**：当前 prompt（用户问题 + 历史对话 + 工具描述 + 上一轮工具返回）、可选的外部状态（时间、位置等）。  
- **决策**：LLM 根据 prompt 生成下一步——可能是“思考”（Reasoning）、或“调用工具 X 且参数为 Y”（Action）、或“最终答案”（Answer）。  
- **执行**：若为工具调用，系统执行工具并把**结果**拼回 prompt；若为最终答案，则返回用户并可能结束回合。  
- **观察**：工具执行结果、错误信息、新状态等，作为下一轮“感知”的一部分。

ReAct、Toolformer、Function Calling 等框架，本质上都是对这一循环的具象化：用固定格式（如 `Thought / Action / Observation`）让 LLM 产出“决策”，由系统负责“执行”并把“观察”反馈回去。

---

## 三、为什么强调“循环”？

- **单次生成**无法应对需要多步工具调用的任务（如先查天气再推荐行程）；循环才能**多步推理与行动**。  
- **观察**使 Agent 能根据真实反馈调整——工具失败可重试或换策略，而不是“闭眼猜到底”。  
- 面试常问：“Agent 和一次调用 LLM 有什么区别？”——关键就是有没有这个**闭环**和**多轮感知-决策-执行**。

---

## 四、小结与面试要点

**小结**：PDA 循环 = 感知（获取环境/工具反馈）→ 决策（LLM 决定下一步）→ 执行（调用工具或输出）→ 观察 → 再感知；是 Agent 多步行动与适应反馈的基础。

**面试要点**：

- 三步：感知（环境与反馈）、决策（下一步做什么）、执行（工具调用或输出）；执行后观察再进入下一轮感知。  
- 在 LLM Agent 中：prompt 含历史+工具描述+上次结果，LLM 生成 Thought/Action/Answer，系统执行工具并把结果拼回 prompt。  
- 与单次 LLM 区别：循环才能多步工具调用和根据反馈调整。

---

## 记忆要点

1. PDA = 感知 → 决策 → 执行 → 观察 → 再感知。  
2. 感知：用户输入、工具返回、历史；决策：LLM 输出下一步；执行：调工具或回复；观察：执行结果。  
3. 循环的意义：多步行动、根据反馈调整，而非一次生成到底。
