# 第 31 题：Agent 的自我反思（Self-Reflection）机制

## 题目

解释 Agent 的自我反思（Self-Reflection）机制。

---

## 一、什么是自我反思？

**自我反思（Self-Reflection）** 指 Agent 在生成输出或执行动作后，**对自身行为与结果进行检视与评估**，并据此**修正策略或输出**。例如：先给出一个答案或执行一系列工具调用，再问自己“结果是否满足目标、是否有错误、是否遗漏步骤”，若发现问题则重新推理、换工具或重试，形成“行动 → 反思 → 改进”的闭环。

在 LLM Agent 中，反思通常由**同一或另一轮模型调用**完成：输入中包含当前轨迹与结果，让模型输出“哪里有问题、应如何改正”，再根据反思结果决定是否重试、回退或补充步骤。

---

## 二、典型实现方式

- **结果校验**：执行完工具或生成答案后，用模型或规则检查“是否达成目标、格式是否正确、是否与观察一致”；若不通过则触发反思与重试。  
- **反思提示**：在 prompt 中显式要求模型对当前步骤或最终答案做自评（如“检查上述推理是否有误”“若结果不对请说明原因并给出修正”），模型输出反思文本，再根据反思决定下一步。  
- **Reflexion 等框架**：将“尝试 → 得到反馈（如测试失败）→ 生成反思（错在哪）→ 把反思存入记忆 → 再尝试”固化为一套流程，用反思文本作为后续尝试的约束或提示，避免重复犯错。  
- **多轮验证**：对关键输出（如代码、决策）多次生成或多次反思，取一致或多数结果，降低幻觉与偶然错误。

---

## 三、作用与注意点

**作用**：减少一次生成带来的错误、遗漏与幻觉；在工具失败或结果异常时能主动纠偏；长期可积累“反思-修正”模式，提升后续表现。

**注意**：反思本身也消耗 token 与调用次数，需在“反思深度/次数”与“延迟、成本”之间权衡；反思质量依赖模型能力，弱模型可能反思不准或重复犯错。

---

## 四、小结与面试要点

**小结**：自我反思 = Agent 对自身行为与结果进行检视与评估并据此改进；实现方式包括结果校验、反思提示、Reflexion 式“尝试-反馈-反思-再试”、多轮验证；用于降低错误与幻觉、支持纠偏与持续改进。

**面试要点**：

- 定义：对自身输出/行为检视与评估，据此修正或重试。  
- 实现：结果校验、反思提示、Reflexion 式流程、多轮验证。  
- 作用：减少错误与幻觉、工具失败时纠偏；需权衡成本与反思深度。

---

## 记忆要点

1. 自我反思 = 对自身行为与结果检视、评估并改进。  
2. 实现：结果校验、反思提示、尝试-反馈-反思-再试（如 Reflexion）、多轮验证。  
3. 作用：降错、纠偏、持续改进；需权衡 token/延迟与反思深度。
