# 第 196 题：投机采样（Speculative Decoding）

## 题目

投机采样（Speculative Decoding）的原理与作用？

---

## 一、原理

**投机采样** = 用**小/快模型**（Draft 模型）**先**连续生成若干 **token**（如 4～8 个），再用**大模型**（Target 模型）**一次**对这批 token **并行验证**（即一次前向算出每个位置在“给定前文”下的**正确**分布）。  
- 若大模型**同意**某位置 token，则**接受**；  
- 若**不同意**，则**拒绝**该 token 及**其后** draft，用大模型**该位置**的采样结果**替换**，并**从该位置**再让 draft 继续“猜”下一段，重复上述过程。  
**效果**：大模型**总前向次数**减少（多步合成“一批”验证）；在 **draft 质量尚可**时，**加速比**可达 **2～3x**，且**分布与原模型一致**（无偏）。

---

## 二、为何能加速

- **大模型**的**瓶颈**常是**每步**都要跑一次**大前向**；**步数**= 生成长度。  
- **投机**把**多步**变成**一批验证**：若 draft 的 **k** 个 token **全被接受**，则这 **k** 步只对应大模型 **1** 次前向；**接受率**越高，**等效步数**越少，**加速**越明显。  
- **Draft** 越准（与小模型能力、任务有关），**接受率**越高；**Draft 太弱**则常被拒，**加速有限**甚至**略慢**（多了一次 draft 的前向）。

---

## 三、实现要点

- **Draft 模型**：**小模型**（同架构小尺寸、或蒸馏）、**同模型量化版**、或**短 context**；需**快**且与**大模型**词表/格式兼容。  
- **验证**：大模型**一次**输入 **prefix + draft 的 k 个 token**，输出**各位置**的 **logits**；与 draft 的 token **逐位比较**（或采样后比较），决定**接受长度**。  
- **长度**：**Draft 长度 k** 影响**加速**与**显存**；k 大则**单次验证**算得多，但**接受率**可能降；常取 **4～8**。  
- **无偏**：在**正确实现**下（接受/拒绝与替换按大模型分布），**生成分布**与**仅用大模型**一致；**质量**不损。

---

## 四、小结与面试要点

**小结**：投机采样用小模型 draft、大模型并行验证，减少大模型前向次数从而加速；接受率越高加速越明显；实现需选 draft、定 k、正确验证逻辑；分布无偏。

**面试要点**：能说清“draft 生成→大模型并行验证→接受/拒绝”的流程；能解释为何能加速（步数合并）；无偏要提到。

---

## 记忆要点

1. 流程：Draft 生成 k 个 token → 大模型一次验证 → 接受/拒绝并替换。  
2. 加速：大模型前向次数减少；接受率越高加速越明显。  
3. 无偏：正确实现下生成分布与原大模型一致。
