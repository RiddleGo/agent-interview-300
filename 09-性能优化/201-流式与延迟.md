# 第 201 题：流式与延迟

## 题目

流式输出如何影响延迟体验？有哪些优化点？

---

## 一、流式对延迟的影响

- **首 token 延迟（TTFT）**：**流式**不改变**实际** TTFT（从请求到首 token 的**计算时间**），但**一旦**首 token 出来就**立刻**推给用户，**感知**上“已经开始回答”，**体验**优于**等整段**再一次性返回。  
- **总时间**：**总生成时间**（到最后一个 token）与**是否流式**无关，由**生成长度**与**每 token 延迟**决定；**流式**不能缩短**总时间**，但**边看边出**感觉更快。  
- **网络**：**流式**需**持续**传小包；**首包**更早发出，**首字**到达用户的时间 ≈ **TTFT + 首包 RTT**；**非流式**要等**整段**再发，**首字**到达 ≈ **总时间 + 一大包 RTT**；流式**显著**改善**首字到达**时间。

---

## 二、优化点

- **降低 TTFT**：**chunked prefill**、**连续批处理**、**KV Cache 复用**、**投机采样**等（见前述题），**实际**缩短首 token 产生时间；流式**放大**其收益。  
- **首包尽快**：**不缓冲**过多 token 再发；**每 token 或每 N token** 即推送；**压缩**（若协议支持）减少**首包**体积与传输时间。  
- **后端到前端**：**推理→网关→前端**整条链**支持流式**（不阻塞、不攒整段）；**CDN/代理**对 **SSE** 等**不缓冲**；**前端**用 **EventSource/ReadableStream** 及时渲染。  
- **弱网**：**弱网**下**小包**易丢或延迟；可**适当**缓冲**若干 token** 再发（权衡**首字**与**流畅度**）；**重连**与**续传**策略（若协议支持）。

---

## 三、小结与面试要点

**小结**：流式不缩短总生成时间，但显著改善首字到达与感知延迟；优化包括降低 TTFT、首包尽快、全链路支持流式及弱网下的缓冲与重连。

**面试要点**：能说清流式改善的是“感知/首字”而非“总时间”；TTFT 与首包、全链路支持要提到。

---

## 记忆要点

1. 流式改善：首字到达与感知，不缩短总时间。  
2. 首字到达 ≈ TTFT + 首包 RTT；降低 TTFT 是关键。  
3. 全链路支持流式、首包尽快、弱网可适当缓冲。
