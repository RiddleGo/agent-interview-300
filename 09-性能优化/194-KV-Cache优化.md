# 第 194 题：KV Cache 优化

## 题目

推理时 KV Cache 的作用与常见优化手段？

---

## 一、KV Cache 是什么

**自回归**生成时，每步只**新算当前 token**，但**注意力**需要**之前所有 token** 的 **Key、Value**。**KV Cache** = 把**已算过的** K、V **存下来**，下一步**只算新 token** 的 K、V，再与**历史 K、V** 拼在一起做 attention，避免**重复计算**。  
**不缓存**则每步都要**重算整段**，**复杂度**从 O(1) 每步变成 O(n)，不可接受**；因此**长序列**推理**必须**用 KV Cache。

---

## 二、内存与带宽

- **内存**：**每层**存 **2 × [batch, head, seq_len, head_dim]**（K 和 V）；**层数×batch×seq** 越大，**显存**越大；**长 context** 时 KV Cache 成为**主要**显存占用。  
- **带宽**：每步要**读**整段 K、V，**写**新 K、V；**带宽**可能成为**瓶颈**（尤其长序列、大 batch）。  
- **优化方向**：**减容量**（量化、稀疏、压缩）、**减读写**（PagedAttention、分层）、**复用**（多请求共享 prefix 的 KV）。

---

## 三、常见优化

- **PagedAttention（vLLM）**：将 **KV Cache** 按**块**管理，**物理块**可**非连续**、**共享**；**相同 prefix** 的请求**共享**同一份 KV 块，**大幅省显存**；**碎片**减少，**利用率**高。  
- **KV Cache 量化**：K、V 用 **INT8** 甚至 **INT4** 存；**省显存**，**略增**计算（反量化）或**精度**略降；可与**权重量化**配合。  
- **Prefix 复用**：**多轮对话**或 **RAG** 中**相同 system/context** 的**前缀**只算一次，**KV 缓存**起来；下次请求**跳过**该 prefix 的 prefill，直接**接续** decode；**显著降 TTFT** 与算力。  
- **稀疏与剪枝**：**长 context** 中**部分** K、V **丢弃**或**压缩**（如只保留**关键位置**）；**精度**有损，需**评估**。  
- **Chunked prefill**：**Prefill** 不一次性算完，**分块**算并**逐步**填入 KV；**首 token** 可**更早**出来，**峰值显存**可控制。

---

## 四、小结与面试要点

**小结**：KV Cache 避免每步重算历史 K、V，但占显存与带宽；优化包括 PagedAttention、量化、Prefix 复用、稀疏与 Chunked prefill；长 context 与多轮/RAG 场景尤其依赖 KV 优化。

**面试要点**：能说清 KV Cache 的作用与为何必要；能举 PagedAttention、Prefix 复用、量化中 1～2 个；长序列显存问题要提到。

---

## 记忆要点

1. 作用：存历史 K、V，每步只算新 token；省重复计算。  
2. 问题：显存与带宽；长 context 时占大头。  
3. 优化：PagedAttention、量化、Prefix 复用、Chunked prefill。
