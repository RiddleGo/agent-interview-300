# 第 255 题：大模型与 Agent 融合

## 题目

大模型与 Agent 如何深度融合？有哪些典型架构与趋势？

---

## 一、融合形态

- **LLM as brain**：大模型作为**推理与决策**中心，负责**理解**指令、**规划**、**调用工具**、**整合**结果并**生成**回复；**记忆**、**检索**、**工具**等作为**外围**模块。  
- **多模型协作**：**专用**模型（**编码**、**检索**、**代码**、**多模态**）与 **LLM** 协同；**路由**或**编排**层决定**调用**谁、**何时**调用。  
- **Agent 能力内化**：**工具使用**、**规划**、**多步推理**通过**预训练/微调**或**提示**写进**单一**或**少模型**；**端到端**可微与**轻量**部署。

---

## 二、典型架构

- **ReAct / 工具调用**：**Thought—Action—Observation** 循环；**LLM** 输出**推理**与**工具**调用，**环境**返回**观测**，再**迭代**直至**答案**。  
- **规划 + 执行**：**高层** **LLM** 做**任务分解**与**计划**；**子任务**由**专用**模型或**工具**执行；**结果**回填**LLM** 做**总结**或**下一步**。  
- **记忆 + 检索**：**长期记忆**（向量库、图）与 **RAG**；**当前**对话与**检索**片段一起**拼进** **prompt**，供 **LLM** 做**上下文**决策。

---

## 三、趋势与面试要点

**趋势**：**更少**模型、**更**统一接口（**多模态**、**多工具**）；**推理**与**工具**更**可控**、**可验证**；**小型化**与**边缘**部署时**大模型 + 小 Agent** 分工。

**小结**：融合形态包括 LLM 为脑、多模型协作、能力内化；典型架构有 ReAct、规划+执行、记忆+检索；趋势是统一接口、可控可验证、大小模型分工。

**面试要点**：能说清「LLM as brain」与外围模块；能画 ReAct 或规划+执行流程；趋势要概括（统一、可控、边缘）。

---

## 记忆要点

1. 融合：LLM 为脑、多模型协作、能力内化。  
2. 架构：ReAct/工具调用、规划+执行、记忆+检索。  
3. 趋势：统一接口、可控可验证、大小模型分工。
