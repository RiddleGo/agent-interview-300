# 第 256 题：长上下文与无限上下文

## 题目

长上下文与「无限」上下文有哪些技术路线与在 Agent 中的应用？

---

## 一、长上下文现状

**长上下文**指模型**一次**可处理的** token 数**从早期 2K/4K 提升到 **32K、128K、1M+**。好处：**少**切段、**少**信息丢失、**长文档**问答、**长对话**与**多轮**工具调用更连贯。

**瓶颈**：**计算**与**显存**随长度**平方**（Attention $O(n^2)$）；**长距离**注意力**质量**下降（**丢失**、**混淆**）；**推理**与**训练**成本高。

---

## 二、技术路线

- **稀疏/线性 Attention**：**Sparse**、**Linear**、**State Space**（**Mamba** 等）将**复杂度**从 $O(n^2)$ 降到 $O(n \log n)$ 或 $O(n)$，从而**支持**更长序列。  
- **外推与插值**：**位置编码** **外推**或**插值**使**短**训**长**推；**质量**在**超**训练长度时**可能**下降，需**评测**。  
- **检索增强**：不**全塞**进上下文，而是**检索**相关**片段**再**拼接**；**无限**感来自**动态**检索，实际**窗口**有限。  
- **层次与摘要**：**分层**表示（**块**级**摘要**再**摘要**）；**顶层**只看**摘要**，**按需**展开；**Memory** 与 **RAG** 结合。

---

## 三、Agent 应用与面试要点

**应用**：**长对话**历史、**多步**任务**状态**、**大**文档 **RAG**、**代码库**级问答；长上下文**减少**分段与**丢失**，**检索**仍常用于**超大**语料。

**小结**：长上下文提升单次处理 token 数；路线有稀疏/线性 Attention、外推/插值、检索增强、层次摘要；Agent 用于长对话、多步与 RAG，检索与长上下文可并存。

**面试要点**：能说清瓶颈（算力、质量）；能举 2 种技术（如 Mamba、检索增强）；Agent 场景要提到长对话与 RAG。

---

## 记忆要点

1. 长上下文：单次 token 数提升；瓶颈在算力与长距离质量。  
2. 路线：稀疏/线性 Attention、外推/插值、检索增强、层次摘要。  
3. Agent：长对话、多步、RAG；检索与长上下文可结合。
