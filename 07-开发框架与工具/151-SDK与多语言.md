# 第 151 题：SDK 与多语言支持

## 题目

LLM/Agent 应用如何提供多语言 SDK？设计要点有哪些？

---

## 一、为何需要 SDK

- **封装**：将 **HTTP/认证/重试/流式** 等细节封装，开发者用少量代码即可调用。  
- **类型与文档**：**类型定义**（TypeScript、Python type hints）与 **文档/注释** 提升体验、减少误用。  
- **多语言**：提供 **Python、JavaScript/TS、Go、Java** 等主流语言 SDK，覆盖不同技术栈。

---

## 二、设计要点

- **API 对齐**：SDK 与 **REST/OpenAI 兼容** 接口一一对应；方法名与参数与 API 文档一致，便于对照。  
- **认证**：统一在**客户端构造**时传入 API Key 或 Token；请求时自动带 Header；支持环境变量与配置文件。  
- **重试与超时**：**可配置**重试次数、退避、超时；对 429、5xx 做重试；流式请求区分连接超时与读超时。  
- **流式**：提供 **async iterator** 或 **callback** 消费 SSE；封装 event 解析与错误处理。  
- **类型**：请求/响应 **DTO** 完整（如 Message、Choice、ToolCall）；便于 IDE 提示与编译期检查。  
- **版本**：SDK 与 **API 版本** 对应；破坏性变更时提供新版本或 deprecated 过渡。

---

## 三、多语言实现

- **Python**：常用 **requests** 或 **httpx**；流式用 **sseclient** 或 **httpx stream**；类型用 **dataclasses** 或 **pydantic**。  
- **JavaScript/TS**：**fetch** 或 **axios**；流式用 **ReadableStream**；类型用 **interface/type** 与 **OpenAPI 生成**。  
- **代码生成**：从 **OpenAPI 规范** 生成各语言客户端，保证与 API 一致；手写部分限于业务逻辑与便捷方法。  
- **统一文档**：API 与各语言 SDK 的**示例、参数说明、错误码**集中维护；可托管在文档站或 GitHub README。

---

## 四、小结与面试要点

**小结**：SDK 封装 HTTP、认证、重试与流式；设计需 API 对齐、认证、重试与超时、类型与版本；多语言可手写或从 OpenAPI 生成；文档与示例需统一维护。

**面试要点**：能说清 SDK 的价值与 2～3 个设计要点（认证、重试、流式、类型）；多语言可提 OpenAPI 生成；版本与文档要提到。

---

## 记忆要点

1. 价值：封装、类型、多语言覆盖。  
2. 要点：API 对齐、认证、重试与超时、流式、类型、版本。  
3. 实现：手写或 OpenAPI 生成；统一文档与示例。
