# 第 138 题：记忆与状态管理

## 题目

在 LangChain / LlamaIndex 中如何实现对话记忆与状态管理？

---

## 一、记忆类型

- **短期**：当前会话的**对话历史**（多轮 user/assistant 消息）；用于维持上下文、指代消解与连贯性。  
- **缓冲**：保留**最近 N 轮**或**最近 K token**（如 `ConversationBufferWindowMemory`、`ConversationTokenBufferMemory`），控制长度与成本。  
- **摘要**：对**更早的历史**做摘要，只把摘要拼入 prompt，节省 token；如 `ConversationSummaryBufferMemory`。  
- **长期**：跨会话的**用户偏好、历史结论**等；常通过向量库或 DB 按需检索注入（见“记忆系统”题）。

---

## 二、在 LangChain 中的使用

- **Memory 组件**：`ConversationBufferMemory`、`ConversationBufferWindowMemory`、`ConversationSummaryBufferMemory` 等，通过 `memory.load_memory_variables({})` 得到 `{"history": "..."}` 等变量，再注入 Prompt。  
- **与链/Agent 结合**：链的 `invoke` 传入 `{"input": "...", "chat_history": [...]}` 或通过 **RunnableWithMessageHistory** 绑定**持久化存储**（如 Redis、DB），按 session_id 存取历史。  
- **消息格式**：Memory 通常输出 `HumanMessage`/`AIMessage` 列表或拼接字符串；Prompt 中预留 `{chat_history}` 占位符。

---

## 三、在 LlamaIndex 中

- **ChatMemory**：LlamaIndex 的 **ChatEngine** 可配 **memory**（如 `SimpleChatStore`）；历史以 Message 形式存储，查询时注入上下文。  
- **与 Index/Context**：长期记忆可通过 **ContextChatEngine** 等，把检索到的节点与历史一起注入；或自定义“从向量库取历史相关片段”的逻辑。

---

## 四、状态持久化

- **会话绑定**：每次请求带 **session_id**；服务端按 session_id 读写历史（内存、Redis、DB）。  
- **过期与清理**：设置 TTL 或最大条数，避免单会话无限增长；敏感信息按策略脱敏或截断后再存。

---

## 五、小结与面试要点

**小结**：记忆分短期（对话历史）、缓冲（窗口/ token 限制）、摘要与长期；LangChain 通过 Memory 组件与 RunnableWithMessageHistory 接入；LlamaIndex 通过 ChatEngine 的 memory 与 Context；状态需 session_id 与持久化存储。

**面试要点**：能说出缓冲、摘要、长期几类记忆；能说清在链/Agent 中如何注入历史（load_memory_variables、占位符）；session 与持久化要提到。

---

## 记忆要点

1. 类型：短期缓冲、窗口/token 限制、摘要、长期（检索）。  
2. LangChain：Memory 组件、load_memory_variables、RunnableWithMessageHistory。  
3. 状态：session_id、持久化（Redis/DB）、过期与清理。
