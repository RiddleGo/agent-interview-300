# 第 143 题：vLLM、TGI 等推理引擎

## 题目

对比 vLLM、TGI（Text Generation Inference）等 LLM 推理引擎的特点与适用场景。

---

## 一、vLLM

- **特点**：**PagedAttention** 将 KV cache 分页管理，提高显存利用率与吞吐；**连续批处理**（Continuous Batching）动态合并请求；支持 **OpenAI 兼容 API**、流式、多 GPU 张量并行。  
- **适用**：**高吞吐、多并发**的在线推理；自托管大批量请求；对延迟要求不极端、更看重 QPS 时。  
- **生态**：与 HuggingFace 模型兼容；支持 Quantization（如 AWQ、GPTQ）。

---

## 二、TGI（Text Generation Inference）

- **特点**：HuggingFace 推出的推理服务；**张量并行**、**Flash Attention**、**流式**；支持 **Server-Sent Events** 与批处理；可 Docker 部署。  
- **适用**：**HuggingFace 模型**首选；需要流式与多 GPU 的中大规模服务。  
- **与 vLLM**：都支持多 GPU、流式与 OpenAI 兼容；vLLM 在吞吐与 PagedAttention 上更突出，TGI 与 HF 生态更紧。

---

## 三、其它

- **TensorRT-LLM**：NVIDIA 生态，**低延迟与高吞吐**优化；需将模型转为 TensorRT 格式；适合对延迟与性能要求极高的场景。  
- **llama.cpp / GGML**：**CPU 或轻量 GPU** 推理，量化支持好；适合边缘或资源受限环境。  
- **SGLang**、**LMDeploy** 等：新兴引擎，各有侧重（如结构化输出、更长上下文、国产芯片适配）。

---

## 四、选型简要

| 需求 | 可考虑 |
|------|--------|
| 高吞吐、自托管 | vLLM |
| HuggingFace 模型、流式 | TGI |
| 极低延迟、NVIDIA 栈 | TensorRT-LLM |
| 边缘/CPU/小模型 | llama.cpp |

---

## 五、小结与面试要点

**小结**：vLLM 以 PagedAttention 与连续批处理见长，适合高吞吐；TGI 与 HF 生态结合紧，支持流式与多 GPU；TensorRT-LLM 偏延迟与 NVIDIA 优化；选型看吞吐、延迟、模型来源与部署环境。

**面试要点**：能对比 vLLM 与 TGI 的 1～2 个核心差异（如 PagedAttention、HF 生态）；能举 1～2 个其它引擎及适用场景。

---

## 记忆要点

1. vLLM：PagedAttention、连续批处理、高吞吐；OpenAI 兼容。  
2. TGI：HF 生态、流式、张量并行；适合 HF 模型。  
3. 其它：TensorRT-LLM 低延迟；llama.cpp 边缘/CPU。
