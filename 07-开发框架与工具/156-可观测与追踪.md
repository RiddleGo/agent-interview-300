# 第 156 题：可观测与链路追踪

## 题目

LLM/Agent 应用的可观测性与链路追踪如何建设？

---

## 一、可观测三支柱

- **日志（Logging）**：**请求级与步骤级**记录；request_id、输入摘要、输出摘要、错误、耗时；**结构化**（JSON）便于检索与解析；集中到 ELK、Loki、云日志等。  
- **指标（Metrics）**：**延迟、QPS、错误率、token 用量**等聚合指标；按接口、模型、租户维度；接入 Prometheus、云监控等；用于告警与容量规划。  
- **追踪（Tracing）**：**分布式 Trace**；一次请求的**完整调用链**（检索→重排→LLM→工具→…）为一条 trace，每步为 span；用 **OpenTelemetry**、**LangSmith** 等；便于定位慢点与错误环节。

---

## 二、LLM/Agent 特有

- **步骤粒度**：除 HTTP 层外，需**业务步骤**（如 prompt 组装、检索、每轮 LLM、工具调用）打 span；记录 token、模型名、检索条数等。  
- **Prompt 与采样**：**敏感信息脱敏**后可选记录 prompt 与 response 摘要，便于 Bad Case 分析；可配置采样率控制体积与成本。  
- **成本与用量**：在 trace 或日志中**关联 token 与成本**，便于按请求/租户归因。  
- **LangSmith / LangFuse**：与 LangChain 等集成，提供**可视化 trace**、**Dataset 与评估**；可自建或用托管。

---

## 三、建设要点

- **统一 request_id**：从网关到检索、LLM、工具**贯穿**同一 id；日志与 trace 可关联。  
- **规范与采样**：**日志格式与字段**统一；高流量时**采样**全量 trace 或详细日志，保证可查又不至于存储爆炸。  
- **告警**：基于指标设**告警**（错误率、P99、预算）；与值班与升级流程打通。

---

## 四、小结与面试要点

**小结**：可观测 = 日志 + 指标 + 追踪；LLM/Agent 需步骤级 span、token 与成本、prompt 采样与脱敏；建设上统一 request_id、规范与采样、告警；可结合 LangSmith 等工具。

**面试要点**：能说清三支柱及在 LLM 场景的延伸（步骤、token、成本）；能举 OpenTelemetry 或 LangSmith；request_id 与采样要提到。

---

## 记忆要点

1. 三支柱：日志、指标、追踪；步骤级 span、token 与成本。  
2. 特有：prompt 采样与脱敏、LangSmith/LangFuse。  
3. 建设：统一 request_id、规范、采样、告警。
