# 第 152 题：插件与扩展机制

## 题目

LLM/Agent 框架中插件与扩展机制如何设计？

---

## 一、插件的含义

**插件** = 在不改主程序的前提下，通过**约定接口**增加能力：如新工具、新数据源、新模型后端、自定义中间件（日志、鉴权、限流）。**扩展** 常指同一机制：可插拔的组件与加载方式。

---

## 二、设计要点

- **接口约定**：定义**统一接口**（如 Tool 的 `name`、`description`、`invoke`；或 Retriever 的 `retrieve(query)`）；插件实现该接口并注册。  
- **发现与加载**：**静态注册**（代码中 import 并 register）；或**动态加载**（从配置/目录扫描、按名加载模块或包）；支持按需启用（白名单、配置开关）。  
- **配置**：插件可有**独立配置**（如 API 端点、参数）；由主程序或配置中心注入，不写死在插件内。  
- **生命周期**：**初始化**（加载配置、建连接）、**就绪**、**关闭**（优雅释放资源）；主程序负责调用 init/close。  
- **隔离与安全**：插件在**沙箱**或受限权限下执行（尤其是执行代码的插件）；校验输入与输出，避免注入与越权。

---

## 三、在 LangChain/LlamaIndex 中的体现

- **LangChain**：**Tool** 即插件；自定义 Tool 注册到 Agent；**Runnable** 可自定义并加入链；**Callback** 可视为“可插拔的观测”。  
- **LlamaIndex**：**Custom Retriever**、**Custom Embedding**、**Custom LLM** 等通过实现基类并注入 **ServiceContext** 或 Index 实现扩展；**Module** 可插拔。  
- **通用**：通过 **entry_points**（Python）、**SPI**（Java）或**配置文件**声明插件，主程序扫描并加载。

---

## 四、小结与面试要点

**小结**：插件 = 约定接口 + 发现与加载 + 配置 + 生命周期；便于增加工具、数据源与中间件；需注意隔离与安全；LangChain/LlamaIndex 的 Tool、Custom Retriever/LLM 等即为扩展点。

**面试要点**：能说清接口约定、发现与加载、配置；能举框架中的扩展点（Tool、Custom Retriever）；安全与隔离要提到。

---

## 记忆要点

1. 设计：接口约定、发现与加载、配置、生命周期。  
2. 框架：LangChain Tool/Runnable/Callback；LlamaIndex Custom Retriever/LLM。  
3. 安全：沙箱、校验、权限限制。
