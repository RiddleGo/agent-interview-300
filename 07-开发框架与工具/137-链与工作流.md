# 第 137 题：链与工作流编排

## 题目

LangChain 中链（Chain）与工作流编排的概念与实现方式。

---

## 一、链（Chain）的概念

**链** = 把多个**步骤**按固定顺序串联：上一步输出作为下一步输入，形成“输入 → 步骤1 → 步骤2 → … → 输出”的流水线。在 LangChain 中，每个步骤通常是 **Runnable**（如 Prompt、LLM、Parser、Tool）；链本身也是 Runnable，可嵌套、复用与组合。

**典型**：RAG 链 = Retriever → 格式化文档 → LLM → 解析；摘要链 = 加载文档 → 分块 → 每块摘要 → 合并摘要。

---

## 二、LCEL（LangChain Expression Language）

**LCEL** 用 `|` 将 Runnable **组合**成链，例如：

```text
chain = prompt | llm | output_parser
```

- 支持**流式**：中间 LLM 流式输出会向上游传递。  
- 支持**批处理**：`chain.batch([...])`。  
- 支持**异步**：`chain.ainvoke(...)`。  
- 便于**绑定**：`chain.with_config(run_name="...")`、`chain.with_fallbacks(...)`。

复杂链可由多条子链用 `|` 或 `RunnableParallel`、`RunnableBranch` 等组合。

---

## 三、工作流与 DAG

当步骤间**非纯线性**（有条件分支、多路并行、循环）时，需**工作流**或 **DAG**：

- **RunnableBranch**：按条件选不同分支（如 query 分类后走不同子链）。  
- **RunnableParallel**：多路并行执行再合并（如多路检索后融合）。  
- **LangGraph**：LangChain 的图式工作流，节点为 Runnable，边为转移条件，支持循环与状态，适合多步 Agent 与复杂编排。

---

## 四、小结与面试要点

**小结**：链 = 多步串联的 Runnable 流水线；LCEL 用 `|` 组合、支持流式与批处理；工作流/DAG 用 Branch、Parallel 或 LangGraph 处理分支与循环；RAG 链、Agent 均为链或图的实例。

**面试要点**：能说清链与 LCEL 的用法；能举例 RAG 链；复杂编排要提到 LangGraph 或 Branch/Parallel。

---

## 记忆要点

1. 链 = 多步串联；每步为 Runnable；链可嵌套。  
2. LCEL：`|` 组合、流式、batch、fallback。  
3. 工作流：Branch、Parallel、LangGraph（图、循环、状态）。
