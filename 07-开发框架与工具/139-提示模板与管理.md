# 第 139 题：提示模板与管理

## 题目

在开发框架中如何管理提示模板（Prompt Template）？最佳实践有哪些？

---

## 一、模板的作用

**提示模板** = 带**占位符**的字符串（或消息列表），在运行时填入变量（如 `{query}`、`{context}`、`{history}`），形成最终 prompt。便于**版本化、复用与 A/B**，避免硬编码长字符串。

---

## 二、在 LangChain 中

- **PromptTemplate**：`PromptTemplate.from_template("...{variable}...")`，`template.format(variable=value)` 或 `prompt.invoke({"variable": value})` 得到最终字符串。  
- **ChatPromptTemplate**：由 **SystemMessagePromptTemplate**、**HumanMessagePromptTemplate** 等组成，支持多角色与多变量；`format_messages(...)` 得到 `Message` 列表。  
- **Few-shot**：**FewShotPromptTemplate** + **ExampleSelector**（如按相似度选示例），动态注入示例。  
- **管理**：模板可放在**代码**、**配置文件**（YAML/JSON）或**外部存储**；通过命名与版本号区分，便于实验与回滚。

---

## 三、在 LlamaIndex 中

- **默认 Prompt**：各模块（Retriever、ResponseSynthesizer、Refine 等）有 **default_prompts**，可通过 **ServiceContext** 或直接覆盖 **get_prompt_from_llm** 等定制。  
- **定制**：在 `ServiceContext` 中设置 `llm_predictor` 的 default prompt，或传入自定义 **PromptTemplate**；管理偏向**模块级默认 + 覆盖**，与 RAG 流程紧密结合。

---

## 四、最佳实践

- **版本与命名**：模板带版本或 id，便于追踪与回滚。  
- **变量集中**：占位符与默认值集中定义，避免拼写与遗漏。  
- **敏感内容**：用户输入、检索结果等用**明确边界**（如 XML 标签）与“仅作数据”的说明，降低注入风险。  
- **可观测**：记录实际使用的模板 id 与变量摘要，便于 A/B 与 Bad Case 分析。

---

## 五、小结与面试要点

**小结**：提示模板 = 占位符 + 运行时填变量；LangChain 用 PromptTemplate/ChatPromptTemplate 与 Few-shot；LlamaIndex 用模块默认 + ServiceContext 覆盖；管理上注重版本、变量集中与安全边界。

**面试要点**：能说清模板的用途与在框架中的用法；版本化与安全（边界、仅作数据）要提到；可举 Few-shot 或多角色模板。

---

## 记忆要点

1. 模板 = 占位符 + 变量；LangChain PromptTemplate/ChatPromptTemplate；LlamaIndex 默认+覆盖。  
2. 管理：版本、命名、变量集中、可观测。  
3. 安全：明确边界、敏感内容仅作数据。
