# 第 154 题：云服务选型

## 题目

LLM/Agent 应用在云服务选型时考虑哪些因素？常见云厂商能力对比。

---

## 一、选型维度

- **模型能力**：**通用与领域模型**的丰富度、版本更新、上下文长度、是否支持 function calling 与多模态；是否支持**微调与专属部署**。  
- **API 与兼容**：是否 **OpenAI 兼容**；**速率限制**、**SLA**；SDK 与文档质量。  
- **数据与合规**：**数据驻留**（是否可指定地域）、**隐私与合规**承诺；是否支持**私有化/专属实例**。  
- **RAG 与生态**：是否提供**向量库、检索、RAG 流水线**；与对象存储、数据库的集成。  
- **成本**：**按 token 或按请求**计费；是否有**预留/包月**优惠；Embedding 与推理分开计费。  
- **运维与支持**：**控制台**、**监控与告警**、**工单与 SLA**；企业支持级别。

---

## 二、常见云厂商简比

- **OpenAI / Azure OpenAI**：模型领先、API 成熟、生态广；Azure 满足企业合规与数据驻留；成本相对高。  
- **AWS Bedrock**：多模型（Claude、Llama 等）、与 AWS 数据与安全集成；适合已深度用 AWS 的客户。  
- **Google Vertex AI**：Gemini 系列、多模态与长上下文；与 BigQuery、Dataform 等集成；适合数据与分析场景。  
- **阿里云百炼 / 通义**：国内合规与数据本地化；多模型与 RAG 能力；适合国内企业与监管要求。  
- **其他**：Anthropic、Cohere、Mistral 等通过 API 或云市场提供；可按模型能力与价格单独选型。

选型时先定**合规与数据**（是否必须国内、是否可上公云），再定**模型与 API**，最后比**成本与生态**。

---

## 三、小结与面试要点

**小结**：选型看模型能力、API 与兼容、数据与合规、RAG 与生态、成本与运维；OpenAI/Azure、AWS、Google、阿里云等各有侧重；先定合规与数据，再定模型与成本。

**面试要点**：能说出 3～5 个选型维度；能举 2～3 个云厂商及特点（如 Azure 合规、AWS 集成、国内用阿里）；数据驻留与私有化要提到。

---

## 记忆要点

1. 维度：模型、API、合规与数据、RAG 与生态、成本、运维。  
2. 厂商：OpenAI/Azure 领先与合规；AWS 集成；Google 数据与分析；阿里国内与合规。  
3. 顺序：先合规与数据，再模型与成本。
