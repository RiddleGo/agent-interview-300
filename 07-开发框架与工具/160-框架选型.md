# 第 160 题：框架选型决策

## 题目

在 LangChain、LlamaIndex、CrewAI、AutoGen 等之间如何做框架选型？

---

## 一、选型维度

- **场景**：以 **RAG/知识库** 为主 → LlamaIndex；以 **通用链与单 Agent** 为主 → LangChain；以 **多角色流水线** 为主 → CrewAI；以 **多 Agent 对话与人机协作** 为主 → AutoGen。  
- **复杂度**：**简单** 对话或 RAG → 低代码（Dify、GPTs）或单框架即可；**复杂** 多步、多 Agent、混合 RAG+Agent → 常需 LangChain + LlamaIndex 或 CrewAI 等组合。  
- **团队与生态**：**Python 为主、需多后端统一** → LangChain；**数据与检索优先** → LlamaIndex；**快速交付、非研发参与** → 低代码平台。  
- **可观测与运维**：是否需要 **LangSmith**、**LangFuse** 等；是否需与企业监控与权限体系集成；影响选型与自研比例。  
- **许可与合规**：**开源许可**（MIT/Apache）是否满足；是否有**商业支持**或**托管版**可选。

---

## 二、组合策略

- **RAG + Agent**：**LlamaIndex** 做索引与 QueryEngine，**LangChain** 做 Agent 与工具封装；QueryEngine 作为 Tool 供 Agent 调用。  
- **多 Agent 流水线**：**CrewAI** 做角色与任务编排，底层 LLM 与工具可用 **LangChain** 组件。  
- **多 Agent 对话**：**AutoGen** 做群聊与人机协作，需要 RAG 时接入 **LlamaIndex** 或 LangChain 的 Retriever。  
- **企业统一栈**：以 **LangChain** 为主做链与 Agent、统一集成与可观测；RAG 部分可 LlamaIndex 或 LangChain 自建；按需引入 CrewAI/AutoGen 做多 Agent。

---

## 三、小结与面试要点

**小结**：选型看场景（RAG vs 链 vs 多 Agent 形态）、复杂度、团队与生态、可观测与合规；常采用组合（LlamaIndex+LangChain、CrewAI+LangChain 等）；先定主场景再选主框架与补充框架。

**面试要点**：能说出 2～3 个选型维度；能举组合策略（如 RAG+Agent 用 LlamaIndex+LangChain）；主场景决定主框架要能说清。

---

## 记忆要点

1. 维度：场景、复杂度、团队与生态、可观测、许可。  
2. 场景→框架：RAG→LlamaIndex；链/单 Agent→LangChain；多角色流水线→CrewAI；多 Agent 对话→AutoGen。  
3. 组合：RAG+Agent、多 Agent 流水线/对话；主场景定主框架。
