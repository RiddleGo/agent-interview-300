# 第 56 题：OpenAI GPTs、LangChain Templates、LlamaIndex 的提示管理方式

## 题目

对比 OpenAI 的 GPTs、LangChain Templates、LlamaIndex 的提示管理方式。

---

## 一、OpenAI GPTs

**方式**：在 GPT 创建/编辑界面中配置 **Instructions**（系统级 prompt）、**Conversation starters**、可选 **Knowledge** 上传；支持多轮对话与 Function Calling。提示以“配置项”形式存在，**版本随 GPT 保存**，可在网页端迭代；不支持代码内细粒度版本与 A/B。

**特点**：产品化、低代码；适合快速搭建专用对话 Agent；提示管理偏“一个 GPT 一套配置”，而非多版本并行与程序化编排。

---

## 二、LangChain Templates

**方式**：提示常以 **PromptTemplate**（含变量占位）形式写在代码或配置中，可与 **ChatPromptTemplate**、**FewShotPromptTemplate** 等组合；支持从文件/环境加载。通过 **LangSmith** 等可做提示的版本记录、对比与追踪。管理偏向**代码即配置**、**链与模板组合**，便于在流水线中复用与切换。

**特点**：模板化、可编程、易与链/Agent 结合；版本与追踪需借 LangSmith 或自建；适合需要复杂链与多步提示的工程化项目。

---

## 三、LlamaIndex

**方式**：提示分散在**各模块**中——如检索、重排序、生成、Agent 的 system prompt 等，可通过 **PromptHelper**、**ServiceContext** 或直接覆盖 **default_prompts** 定制；也支持从文件加载。管理偏向**模块级默认 + 覆盖**，与 RAG/Agent 流程紧密结合，但统一“提示中心”不如 LangChain 模板集中。

**特点**：与 RAG/检索/生成流程深度集成；按模块定制；适合以 RAG 或检索为核心的应用，提示管理随模块走。

---

## 四、对比小结

| 维度 | GPTs | LangChain | LlamaIndex |
|------|------|-----------|------------|
| **形态** | 产品配置（Instructions 等） | 代码模板 + 可选 LangSmith | 模块默认 + 覆盖、文件 |
| **版本与追踪** | 随 GPT 保存 | LangSmith/自建 | 多为自建 |
| **适用** | 快速产品、单 Agent | 复杂链、多步、工程化 | RAG/检索中心、模块化 |

面试可答：GPTs 是产品内配置、低代码；LangChain 是代码模板+链、可接 LangSmith；LlamaIndex 是模块级默认与覆盖、偏 RAG 流程；按项目形态选——快速产品用 GPTs，复杂链用 LangChain，RAG 中心用 LlamaIndex。

---

## 再深入一层：工程上还要对齐「Prompt Hub」与治理

- **OpenAI / 平台侧**：除 GPTs 外，**Assistants API**、**Stored Prompts**、**Responses API** 等把「提示当资源」管理：版本、变量、与工具/文件绑定；面试可补充「产品配置 → API 资源」这条线。  
- **LangChain**：除 LangSmith 外，社区有 **LangChain Hub** 拉取共享提示；企业内网通常自建 **私有 Hub + MR 审批**，避免提示散落仓库。  
- **LlamaIndex**：提示与 **query engine / chat_engine** 绑定深；升级库版本时要关注 **default prompt 变更** 带来的静默行为漂移，需锁定版本或显式覆盖。

## 常见追问

- **三家能混用吗？** 能：例如 LC 编排 + LI 做索引 + 商用 API 的 system prompt 存配置中心；关键是**单一事实来源**（谁拥有最终 system 文本）。  
- **如何做权限？** GPTs 靠平台账号；代码侧靠 **Git + CODEOWNERS** 与运行时 **只读密钥** 注入敏感段。  
- **A/B 在哪一层做？** 可在网关按 **prompt_version** 分流，与框架解耦。

---

## 五、小结与面试要点

**小结**：GPTs 以产品配置管理提示；LangChain 以模板+链+可选 LangSmith 管理；LlamaIndex 以模块默认与覆盖管理、偏 RAG；三者形态与适用场景不同。进阶上要联系 API 化提示资源、私有 Hub、库升级导致的默认 prompt 漂移与统一治理。

**面试要点**：

- GPTs：Instructions 等配置、产品化、低代码。  
- LangChain：PromptTemplate、链组合、LangSmith 追踪。  
- LlamaIndex：模块默认与覆盖、与 RAG 流程集成。  
- 选用按项目：快速产品 vs 复杂链 vs RAG 中心。  
- 进阶：Assistants/Stored Prompts、私有 Hub、版本锁定、网关级 A/B。

---

## 记忆要点

1. GPTs：产品配置（Instructions），低代码。  
2. LangChain：模板+链，LangSmith 追踪。  
3. LlamaIndex：模块默认与覆盖，偏 RAG。
