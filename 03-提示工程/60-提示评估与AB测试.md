# 第 60 题：评估提示效果的指标与如何设计提示 A/B 测试

## 题目

评估提示效果的指标有哪些？如何设计提示 A/B 测试？

---

## 一、常用评估指标

- **任务相关**：**准确率/ F1**（分类、抽取）、**ROUGE/BLEU**（摘要、翻译）、**任务成功率**（如 Agent 任务是否完成）、**通过率**（如代码/数学题）。  
- **格式与合规**：**格式通过率**（JSON/XML 是否合法、必填字段是否齐全）、**安全通过率**（是否触发安全拒绝、是否含违规内容）。  
- **体验与效率**：**平均轮数/步数**（少则好）、**平均 token 数**（成本与延迟）、**用户满意度**（评分、点赞、负反馈率）。  
- **稳定性**：同一输入多次调用的**一致率**或**方差**；Bad Case 比例与类型分布。  
- **业务**：转化率、解决率、人工接管率等，按业务定义。

按任务类型选主指标（如 QA 看准确率、摘要看 ROUGE、Agent 看任务成功率），再辅以格式、安全、成本与体验指标。

**LLM-as-Judge 注意**：用作自动指标时要防**位置偏见**（先出现的答案被偏好）、**自嗨打分**（裁判模型与生成模型同源）；应用**交换顺序双评**、异构裁判、或人类校准集。

---

## 二、如何设计提示 A/B 测试

- **明确变量**：只对**单一或少量 prompt 维度**做 A/B（如仅改指令、仅改 few-shot 示例、仅改格式），便于归因。  
- **分流**：按 **用户 id 或 request id** 稳定分流到 A/B（如 hash 取模），保证同一用户/会话始终看到同一版本，避免体验混乱。  
- **样本量**：根据主指标方差与期望提升幅度估算**最小样本量**，跑满后再下结论；避免过早停止。  
- **多版本**：可 A/B/C 或多臂，同时比较多个 prompt；需校正多重比较（如 Bonferroni）或使用序贯检验。  
- **正交与分层**：若同时测“提示”与“模型/参数”，可做**正交实验**或分层分析，分离提示效果与模型效果。  
- **监控与回滚**：实时监控各版本的主指标与安全/格式指标，异常版本自动降流或回滚。  
- **记录与复现**：记录每请求的**版本 id、分流标签、输入与输出采样**，便于复现与 Bad Case 分析。

---

## 再深入一层：A/B 里最容易被忽略的统计与产品效应

- **辛普森悖论**：全局上 A 胜 B，但按国家/端型分层后反转；要做 **分层报告** 与足够样本 per 层。  
- **新奇效应**：新提示短期互动率更高，长期回落；可设 **暖机期后** 再读主指标，或看留存/次日满意度。  
- **网络效应**：对话产品里用户会**适应**新口吻，需**跨会话**指标而非仅首轮。

## 常见追问

- **sequential testing 还要固定样本量吗？** 可用序贯检验提前停，但要控制 **alpha 膨胀**，避免「看着好就停」。  
- **多指标怎么决策？** 先定 **guardrail**（安全、P99 延迟）一票否决，再在主业务指标上做加权或 Pareto。  
- **离线评测 vs 在线 A/B？** 离线便宜但分布漂移；在线贵但真实；常见：**离线筛候选 → 小流量在线 → 全量**。

---

## 三、小结与面试要点

**小结**：指标包括任务准确率/成功率、格式与安全通过率、轮数/token、满意度、稳定性与业务指标；A/B 设计需明确变量、稳定分流、足够样本、可选多版本与正交、监控回滚与记录复现。进阶上要谈 LLM 评委偏见、辛普森/新奇效应与序贯检验。

**面试要点**：

- 指标：任务（准确率、成功率）、格式与安全、成本与体验、稳定性、业务。  
- A/B：单变量、稳定分流、样本量、多版本与正交、监控回滚、记录复现。  
- 主指标按任务选，辅以格式、安全、成本。  
- 进阶：LLM-as-Judge 去偏、分层与新奇效应、guardrail 多指标、离线在线流水线。

---

## 记忆要点

1. 指标：任务准确率/成功率、格式与安全、token/轮数、满意度、稳定性、业务。  
2. A/B：单变量、稳定分流、样本量、监控回滚、记录复现。  
3. 主指标按任务选，辅以格式、安全、成本。  
4. LLM 评委去偏；辛普森/新奇效应；guardrail 多指标与离线→在线流水线。
