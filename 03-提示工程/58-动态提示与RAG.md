# 第 58 题：动态提示（Dynamic Prompting）在 RAG 系统中的应用

## 题目

解释动态提示（Dynamic Prompting）在 RAG 系统中的应用。

---

## 一、什么是动态提示？

**动态提示** 指 prompt 的**部分内容在请求时按上下文生成**，而非完全静态写死。在 RAG 中常见：根据**当前查询**检索到的文档、重排序结果、用户画像、会话历史等，**实时拼成**“检索到的段落”“当前问题”“历史摘要”等，再与固定模板组合成最终 prompt。

---

## 二、在 RAG 中的应用

- **检索结果注入**：根据用户 query 检索到的 **top-k 片段**（或经重排序后的子集）动态填入 prompt 的“上下文”区；query 变则检索结果变，prompt 随之变。  
- **查询改写与扩展**：先对原始 query 做**改写/扩展**（如多查询、假设性文档嵌入 HyDE），用改写后的 query 检索，再把**原始 query + 检索结果**填入 prompt；动态部分=改写后的检索与最终上下文。  
- **历史与会话**：多轮 RAG 中，将**当前轮 query + 历史轮摘要或关键句**动态填入，避免全文历史占满 context。  
- **模板选择**：根据** query 类型**（事实/比较/总结）或**检索质量**（有无高相关结果）选择不同 prompt 模板或不同“无检索时”的降级话术，也属动态提示。  
- **长度与裁剪**：根据**剩余 context 预算**动态决定保留多少检索结果、是否做摘要，保证不超长。

---

## 三、小结与面试要点

**小结**：动态提示 = 请求时按上下文生成 prompt 部分内容；在 RAG 中用于检索结果注入、查询改写后的检索与上下文、历史与会话、模板选择、长度与裁剪等，使 prompt 随 query 与检索结果变化。

**面试要点**：

- 动态提示：请求时按上下文生成部分 prompt。  
- RAG 中：检索结果注入、查询改写与检索、历史会话、模板选择、长度裁剪。  
- 目的：让 prompt 随 query 与检索结果适配，提高相关性与可控性。

---

## 记忆要点

1. 动态提示 = 请求时按上下文生成部分 prompt。  
2. RAG 中：检索结果注入、改写与检索、历史、模板选择、长度裁剪。  
3. 目的：prompt 随 query 与检索结果适配。
