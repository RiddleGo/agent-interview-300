# 第 57 题：多模态提示（Multimodal Prompting）及设计要点

## 题目

什么是多模态提示（Multimodal Prompting）？设计要点是什么？

---

## 一、什么是多模态提示？

**多模态提示** 指 prompt 中除**文本**外还包含**其他模态**的输入（如图像、音频、视频），以及可选的**跨模态指令**（如“根据上图回答问题”“描述这段音频”）；模型需同时理解文本与这些模态，并生成符合要求的输出。多模态大模型（如 GPT-4V、Gemini、Claude 3）支持“文本+图像”等组合，多模态提示即针对这类能力的设计。

---

## 二、设计要点

- **模态顺序与位置**：明确“先图后文”“先文后图”或“文图交错”在模型中的效果；通常图像与相关说明放在一起，避免图与题不对应。  
- **指令清晰**：明确要模型“看哪里、做什么”——如“根据第一张图…”“比较图1与图2”；避免模糊的“根据图片”。  
- **分辨率与裁剪**：大图可能被压缩或分块，关键区域可**先裁剪再送入**或指明“关注图中红框”；长视频需摘要或关键帧。  
- **文本描述补充**：对图/视频做简短**文本描述**（如 OCR 结果、物体列表）再拼入 prompt，可加强模型对细节的利用。  
- **格式与 API**：按模型要求传图（base64、URL、多图顺序）；多图时标清顺序与对应问题。  
- **安全与边界**：图像也可能含敏感信息或用于越狱；需与文本一样做内容审核与注入防范。

---

## 再深入一层：多模态的「对齐税」更高

- **OCR 与布局**：表格、票据类图像，纯视觉编码易丢**单元格对齐关系**；常用「图 + 结构化 OCR / HTML 表」双通道输入，并提示模型**以表格为准**。  
- **时序与视频**：长视频要**关键帧选择策略**（场景切分、字幕对齐、运动检测），否则模型看到的是「幻灯片」而非过程。  
- **模态冲突**：当 OCR 文本与像素内容矛盾（PS、遮挡）时，要在指令里规定**仲裁优先级**（例如「以可见像素为准，OCR 仅供参考」）。

## 常见追问

- **多图顺序搞反了怎么办？** 在文本里显式编号「图 A / 图 B」，与 API 传图顺序一一对应；必要时在图上叠**角标**。  
- **如何省 token？** 先小分辨率路由分类，再只对 ROI **高清二次裁剪**；或对背景区域打码降信息。  
- **多模态越狱？** 图中藏违规字、二维码链到恶意页；需**视觉内容安全**与外链策略。

---

## 三、小结与面试要点

**小结**：多模态提示 = prompt 中含图像/音频/视频等+文本及跨模态指令；设计要点包括模态顺序与位置、指令清晰、分辨率与裁剪、文本补充、格式与 API、安全与边界。进阶上要谈 OCR/布局、视频关键帧、模态冲突仲裁与多模态安全。

**面试要点**：

- 定义：含非文本模态（图/音/视频）的 prompt + 跨模态指令。  
- 要点：顺序与位置、指令清晰、分辨率与裁剪、文本补充、格式、安全。  
- 适用模型：GPT-4V、Gemini、Claude 等多模态大模型。  
- 进阶：OCR+视觉双通道、关键帧、冲突仲裁、多图编号、视觉安全。

---

## 记忆要点
