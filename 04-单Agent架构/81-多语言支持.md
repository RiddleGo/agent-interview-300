# 第 81 题：多语言支持

## 题目

单 Agent 如何支持多语言？设计与实现上有哪些常见做法？

---

## 一、多语言涉及哪些层面？

**输入**：用户用不同语言提问（中、英、日等），Agent 需正确理解意图与实体。**输出**：回复语言需与用户偏好或系统配置一致（如界面语言、回答语言）。**内部**：规划、工具调用、错误信息与日志可能涉及多语言；工具与下游 API 也可能返回多语言内容。设计目标是在**识别语言、统一处理与按需输出**之间保持一致体验。

---

## 二、常见实现方式

**语言识别**：对用户输入做**语言检测**（如 langdetect、或 LLM 输出 language code），用于后续路由或 prompt 约束。**模型与 prompt**：使用**多语言模型**（如多语预训练 LLM），在 system prompt 中约定“根据用户语言回复”或“始终用中文回复”；关键文案与工具描述可做多语资源表，按语言加载。**工具与下游**：若工具返回多语言，可在解析后统一转成目标语言或保留原文；对外 API 支持 `Accept-Language` 或 `lang` 参数。**本地化**：前端与固定文案（按钮、错误提示、帮助）用 i18n 资源包，按用户语言或地域切换。

---

## 三、注意点

**一致性**：同一会话内尽量保持一种主要语言，避免中英混杂（除非用户明确切换）。**语种与质量**：小语种可能效果较差，可标注“当前最佳支持语言”或降级策略。**成本与延迟**：多语言模型或额外翻译步骤会带来成本与延迟，需权衡；可对高频语言做优化、长尾语言走通用路径。

---

## 小结与面试要点

**小结**：多语言覆盖输入理解、输出语言与内部处理；实现包括语言检测、多语模型与 prompt 约定、工具与 API 的 lang 参数、前端 i18n；需注意会话内语言一致、小语种质量与成本权衡。

**面试要点**：语言检测+多语模型+prompt 约定；工具与 API 的 lang；i18n 与一致性、小语种与成本。

---
