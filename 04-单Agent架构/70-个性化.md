# 第 70 题：个性化（Personalization）与用户画像

## 题目

单 Agent 如何实现个性化？用户画像如何构建与使用？

---

## 一、个性化的目标与场景

**个性化**指根据**用户身份、偏好、历史行为与上下文**调整 Agent 的回复风格、推荐内容、默认参数或决策偏好，使体验更贴合个体。典型场景：回复简洁/详细、语言风格、领域偏好（如技术/商务）、常用工具或默认选项、敏感操作确认级别等。实现依赖**用户画像**与**会话内上下文**的结合。

---

## 二、用户画像的构建与存储

**数据来源**：显式（用户设置、问卷、反馈）与隐式（历史对话、点击、任务完成情况、工具使用习惯）。**画像内容**：人口属性、偏好标签（领域、风格、语言）、行为统计（常用功能、平均轮次）、风险/权限级别等；可结构化（键值/表）或向量化（偏好描述 embedding 做相似推荐）。**存储**：与用户 ID 绑定，存关系型库或 NoSQL；更新方式可为实时写入或离线批处理+定期同步；需注意**隐私与合规**（最小必要、脱敏、可删除）。

---

## 三、在 Agent 中的使用

画像在**规划与生成**阶段注入：在 system 或 context 中加入“用户偏好摘要”（如“偏好简短回答、常用搜索”），或作为检索条件从长期记忆中召回相关片段；**工具选择与参数**可根据画像设默认（如默认地域、语言）；**权限与安全**可结合用户级别做差异化策略。注意**冷启动**（新用户无画像时用默认或引导收集）与**画像新鲜度**（定期更新、会话内临时覆盖）。

---

## 小结与面试要点

**小结**：个性化通过用户画像（显式+隐式构建，存结构化/向量）与会话上下文结合，在回复风格、推荐、默认参数与权限上做差异化；需兼顾冷启动、新鲜度与隐私合规。

**面试要点**：画像=显式+隐式、结构化+向量；注入 prompt/规划与工具默认；冷启动、更新与隐私。

---

## 记忆要点

1. 个性化：按用户身份、偏好、历史调整风格、内容与决策；依赖画像+会话上下文。  
2. 画像构建：显式设置+隐式行为；内容=属性、偏好、行为统计；存储与用户 ID 绑定，注意隐私。  
3. 使用：注入 prompt、默认工具/参数、权限分级；处理冷启动与画像更新。
