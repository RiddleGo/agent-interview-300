# 第 68 题：ReAct、Reflexion、LATS 演进关系

## 题目

简述 ReAct、Reflexion、LATS 的演进关系与各自核心思想。

---

## 一、ReAct（Reasoning + Acting）

**ReAct** 将**推理**与**行动**交织：Agent 交替产生“Thought（思考）— Action（工具调用）— Observation（观察结果）”，用观察反馈驱动下一步思考，形成“思考→行动→观察”的循环。核心是**把推理显式化**并和工具调用绑定，避免纯 CoT 的幻觉与工具脱节，适合需要多步工具调用的问答与决策任务。

---

## 二、Reflexion（反思与经验记忆）

**Reflexion** 在 ReAct 基础上增加**自我反思与经验记忆**：任务失败或达到检查点时，让 Agent 总结“哪里错了、为何错、应如何改”，将反思结果作为**经验**存入长期记忆；下次类似任务时优先参考这些经验，从而减少重复错误。核心是**从失败中学习**并显式复用，提升多轮或多任务下的样本效率与鲁棒性。

---

## 三、LATS（LLM Agents as Tree Search）

**LATS** 将 Agent 决策过程建模为**树搜索**：每个节点是一次 LLM 调用（可产生多个候选动作或推理分支），通过**价值模型**或**启发式**对节点评分，按 Best-First 等策略扩展与回溯，最终选出一条从根到叶的最优轨迹。核心是**把单条轨迹扩展为搜索空间**，用搜索+价值评估替代“一次生成一条链”，适合对正确性要求高、需要回溯与剪枝的复杂任务。

---

## 小结与面试要点

**小结**：ReAct=推理与行动交织的 Thought-Action-Observation 循环；Reflexion=在 ReAct 上加入反思与经验记忆，从失败中学习；LATS=将决策建模为树搜索，用价值模型与搜索策略求更优轨迹。三者是“单链→带记忆→搜索”的演进。

**面试要点**：ReAct=显式推理+工具循环；Reflexion=反思+经验记忆；LATS=树搜索+价值评估；演进=单链→记忆→搜索。

---

## 记忆要点

1. ReAct：Thought-Action-Observation 循环，推理与行动交织，减少幻觉与工具脱节。  
2. Reflexion：反思总结错误与改进，经验记忆复用，从失败中学习。  
3. LATS：决策为树搜索，价值模型评分，Best-First 等策略，回溯与剪枝求优轨迹。
