# 第 65 题：自我反思实现（反思触发与改进机制）

## 题目

单 Agent 中如何实现自我反思？反思的触发条件与改进机制有哪些常见做法？

---

## 一、反思要解决什么？

Agent 在执行中可能产生错误推理、选错工具或遗漏步骤。**自我反思**指在关键节点或失败后，让 Agent 对自身输出进行审视（如“上一步结论是否合理？”“工具结果是否支持当前结论？”），并据此修正计划或重试，从而提升决策质量与鲁棒性。

---

## 二、反思触发与改进机制

**触发条件**常见有：**固定节点**（每 N 步或每轮工具调用后）、**异常驱动**（工具报错、超时、返回空）、**置信度/一致性检查**（模型自评分数低或与历史矛盾）、**用户/规则校验**（关键动作前由规则或用户确认）。

**改进机制**：用**反思 prompt** 让 LLM 总结“哪里错了、为何错、应如何改”，将反思结果与原始轨迹一起注入下一轮规划或重试；可结合 **Reflexion** 式“经验记忆”（把失败与修正存为教训，后续类似任务优先参考）；对工具失败可触发**重试、换工具或降级策略**。

---

## 三、实现注意点

反思会增加延迟与调用次数，需在**触发频率**与**收益**之间权衡；反思 prompt 要明确“反思范围”（仅上一步 vs 整段轨迹）和“输出格式”（自然语言总结 vs 结构化修正建议）；可与**决策质量评估**结合，仅在对质量存疑时触发深度反思。

---

## 小结与面试要点

**小结**：自我反思通过触发条件（固定节点、异常、置信度、校验）与改进机制（反思 prompt、经验记忆、重试/降级）提升 Agent 正确性与鲁棒性；需权衡触发频率与成本。

**面试要点**：触发=节点/异常/置信度/校验；改进=反思 prompt+经验记忆+重试；注意成本与反思范围、格式的设计。

---

## 记忆要点

1. 反思触发：固定节点、异常驱动、置信度/一致性、用户/规则校验。  
2. 改进机制：反思 prompt 总结错误与修正，经验记忆（Reflexion），重试/换工具/降级。  
3. 实现需权衡：触发频率 vs 延迟与调用成本；明确反思范围与输出格式。
