# 第 79 题：缓存策略（常见查询缓存）

## 题目

单 Agent 中常见缓存策略有哪些？如何对“常见查询”做缓存设计？

---

## 一、为什么要缓存？

Agent 的 LLM 调用与部分工具调用（如通用知识检索、天气、汇率）成本高或延迟大，且**相同或相似请求**可能重复出现。对这类请求做**缓存**可降低延迟、节省成本、减轻下游压力。设计要点：**缓存什么**（完整回复、中间结果、工具返回值）、**以什么为 key**（原始 query、query 归一化、embedding 相似）、**失效与更新**（TTL、版本、失效策略）。

---

## 二、常见缓存策略

**精确匹配**：以原始 query 或归一化后的 query（去空格、小写等）为 key，命中则直接返回缓存回复或工具结果；实现简单，命中率依赖 query 重复度。**语义相似**：对 query 做 embedding，用向量检索找相似历史 query，若相似度超过阈值则复用其缓存；适合表述多样但意图相同的场景；需设定阈值与索引更新。**分层缓存**：热点用本地/进程内缓存，全量用 Redis 等分布式缓存；或“模型结果缓存”与“工具结果缓存”分开，按需失效。**TTL 与失效**：根据数据新鲜度设 TTL（如天气 1 小时、知识库 1 天）；关键配置或数据变更时主动失效相关 key。

---

## 三、与 Agent 的配合

可在**规划前**查缓存：若命中“完整问答”则直接返回，不再调用模型与工具；或仅缓存**工具结果**，规划照常但工具层返回缓存。需注意**隐私与隔离**：按用户或租户隔离缓存，避免跨用户泄露；敏感 query 可不缓存或脱敏后缓存。**监控**：缓存命中率、延迟与成本节省，便于评估与调参。

---

## 小结与面试要点

**小结**：缓存用于降低延迟与成本、减轻下游压力；策略包括精确匹配、语义相似、分层与 TTL；可与规划前或工具层配合，需注意隔离与隐私、失效策略及命中率监控。

**面试要点**：key=精确 query 或语义相似；缓存内容=完整回复或工具结果；TTL 与失效、按用户隔离、命中率监控。

---

## 记忆要点

1. 目的：降延迟、省成本、减下游压力；缓存完整回复或工具结果。  
2. 策略：精确 key、语义相似 key、分层缓存、TTL 与主动失效。  
3. 注意：用户/租户隔离、敏感不缓存、命中率与效果监控。
