# 第 71 题：安全沙箱与工具权限

## 题目

单 Agent 中如何通过安全沙箱与工具权限控制风险？

---

## 一、为什么需要沙箱与权限？

Agent 能调用外部工具（执行代码、访问文件、调用 API），若不对**执行环境**与**可调用范围**做限制，可能发生越权、数据泄露、系统破坏或滥用。**安全沙箱**隔离执行环境，限制资源与能力；**工具权限**控制“谁在何种条件下可调用哪些工具、带何种参数”，二者结合可把风险控制在可接受范围内。

---

## 二、安全沙箱常见做法

**进程/容器隔离**：工具在独立进程或容器（如 Docker、gVisor）中执行，限制 CPU、内存、网络与文件系统访问；执行完毕即销毁或复用干净实例。**受限运行时**：如仅允许在受限解释器或 VM 中执行代码，禁止危险系统调用与文件路径。**网络与 I/O 限制**：白名单域名/IP、禁止内网或敏感端点、限制读写路径。**超时与资源上限**：单次执行超时、最大内存与 CPU，防止死循环或资源耗尽。沙箱策略需与**业务敏感度**匹配：高敏感操作（如删库、支付）需更严隔离与审计。

---

## 三、工具权限设计

**按角色/用户**：不同用户或 API Key 拥有不同工具集合（白名单）；敏感工具需额外权限或审批。**按参数校验**：对工具参数做 schema 与业务规则校验（如禁止 `rm -rf /`、限制查询范围）；对输出做脱敏或截断。**审计与熔断**：记录“谁在何时调用了何工具与参数”，异常调用触发熔断或人工复核。与**身份与认证**结合：请求带用户/会话身份，权限中间件在调用前校验并可能拒绝或降级。

---

## 小结与面试要点

**小结**：安全沙箱通过进程/容器隔离、受限运行时、网络与 I/O 限制、超时与资源上限，限制工具执行环境；工具权限通过角色/用户白名单、参数校验与审计/熔断控制“谁能调什么、怎么调”，二者结合降低 Agent 工具调用风险。

**面试要点**：沙箱=隔离+资源与网络限制+超时；权限=角色白名单+参数校验+审计熔断；需与身份认证结合。

---

## 记忆要点

1. 沙箱目的：隔离执行环境，限制资源、网络与文件访问，防止越权与滥用。  
2. 手段：进程/容器隔离、受限运行时、网络白名单、超时与资源上限。  
3. 工具权限：按角色/用户白名单、参数校验、审计与熔断；与身份认证结合。
