# 第 78 题：A/B 测试框架

## 题目

单 Agent 场景下如何设计 A/B 测试框架？关键环节有哪些？

---

## 一、为什么要 A/B 测试？

Agent 有大量可调因素：模型版本、prompt、规划策略、工具组合、超参（温度、最大步数）等。仅凭经验难以判断哪种配置更优。**A/B 测试**通过将流量按比例分到不同策略（A/B 或多组），在相同时间段内对比**指标**（完成率、满意度、延迟、成本），做统计显著性检验，支撑决策与迭代。

---

## 二、关键环节

**分流**：按用户 ID、session ID 或 request ID 做稳定哈希分流，保证同一用户/会话始终命中同一组，避免体验抖动；比例可配置（如 50% A / 50% B）。**实验配置**：定义实验名称、分组、各组的策略参数（如 model_id、prompt_version、max_steps）；配置中心或数据库管理，支持灰度与回滚。**指标采集**：在各组内采集相同指标（任务完成率、错误率、延迟、token 消耗、用户反馈）；需带 experiment_id、group 等维度便于后续分析。**统计与决策**：对指标做显著性检验（如 t 检验、置信区间），达到显著性且稳定后决定是否全量；注意**多重比较**与**实验时长**（避免过早结论）。

---

## 三、与 Agent 的衔接

Agent 入口根据分流结果加载对应配置（模型、prompt、规划器参数），全链路打上实验标签；日志与指标系统按实验分组聚合。可支持**分层实验**（如模型层与 prompt 层正交），避免互相干扰。新策略上线前建议先小流量 A/B，再逐步放量。

---

## 小结与面试要点

**小结**：A/B 测试通过稳定分流、实验配置、指标采集与统计检验，对比不同 Agent 策略的效果；关键环节是分流稳定性、配置管理、指标维度与显著性分析；可与分层实验结合。

**面试要点**：分流=用户/session 稳定哈希；配置=分组与参数；指标带实验维度；统计显著性+实验时长；与 Agent 配置与日志衔接。

---

## 记忆要点

1. 目的：对比不同策略（模型、prompt、规划等）的效果，做数据驱动决策。  
2. 环节：稳定分流、实验配置、指标采集（带 experiment/group）、统计检验与决策。  
3. 注意：同一用户固定组、多重比较、实验时长；可与分层实验结合。
