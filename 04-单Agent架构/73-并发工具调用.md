# 第 73 题：并发工具调用（并行 vs 串行）

## 题目

单 Agent 何时应并行调用多个工具、何时应串行？设计与实现上要注意什么？

---

## 一、并行 vs 串行的适用场景

**串行**：后续步骤**依赖**前一步结果（如先搜索再根据结果计算、先鉴权再调业务接口）。必须等上一步完成并解析结果后再决定下一步动作与参数，否则无法保证正确性。**并行**：多个工具调用**彼此独立**、无数据依赖（如同时查天气、查新闻、查日历），可同时发起以降低总延迟、提升吞吐。判断依据主要是**数据依赖图**：若调用 A 的输出是 B 的输入，则 A→B 串行；若 A、B 无依赖，可并行。

---

## 二、设计与实现要点

**依赖分析**：由规划模块或 LLM 输出步骤时标注依赖（或从自然语言中解析），构建 DAG；无依赖的节点可并行执行。**并发控制**：用线程池/异步或 goroutine 等并发执行独立调用；设置**整体超时**与**单工具超时**，避免一个慢调用拖死整批。**结果聚合**：并行完成后按原规划顺序或依赖关系聚合结果，再注入下一轮推理或生成；若部分失败，需定义策略（全部重试、部分降级、报错）。**限流与资源**：并行会放大 QPS 与下游压力，需限流、熔断与背压，避免打爆工具或自身。

---

## 三、与 ReAct/规划的配合

在 ReAct 或 LLM 规划中，可约定“本步可并行调用的工具列表”及“下一步依赖本步哪些结果”；执行层解析后并发执行独立调用，再统一 Observation 回传。也可用**静态分析**（工具描述中声明输入输出）辅助生成 DAG，减少对 LLM 依赖。

---

## 小结与面试要点

**小结**：无依赖时可并行调用以降低延迟，有依赖则串行；实现需依赖分析/DAG、并发执行、超时与结果聚合，并注意限流与资源；可与规划/ReAct 配合约定并行与依赖。

**面试要点**：并行=无数据依赖；串行=有依赖；实现=DAG+并发+超时+聚合+限流。

---

## 记忆要点

1. 并行：多工具无数据依赖，同时发起降延迟；串行：有依赖则按序执行。  
2. 实现：依赖分析/DAG、并发执行、整体与单工具超时、结果聚合与部分失败策略。  
3. 注意：限流与下游压力；与规划/ReAct 约定并行与依赖关系。
