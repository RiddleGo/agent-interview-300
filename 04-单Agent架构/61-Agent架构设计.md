# 第 61 题：具备工具使用能力的 Agent 系统架构设计

## 题目

设计一个具备工具使用能力的 Agent 系统，画出架构图并解释关键模块。

---

## 一、整体架构（概念图）

```
用户输入 → [ 编排层 ] → [ LLM ] → 解析 → [ 工具层 ] → 执行 → 观察 → 回填 → [ LLM ] → …
                ↑                                    ↓
           [ 记忆/状态 ]  ←———————————————  工具结果 / 错误
```

**编排层**：接收用户请求，维护会话/任务状态，组装 prompt（系统指令、工具描述、历史、当前输入），调用 LLM，解析输出并路由到工具或最终回答。  
**LLM**：生成推理（Thought）、工具调用（Action）或最终答案（Answer）。  
**解析层**：从模型输出中识别 Action（工具名+参数）或 Answer，校验格式与工具是否在允许列表。  
**工具层**：注册的工具列表（名称、描述、参数 schema、执行函数），根据解析结果执行并返回 Observation 或错误。  
**记忆/状态**：短期（当前会话的对话与工具链）、可选长期（用户偏好、历史摘要），供编排层注入 prompt。

---

## 二、关键模块说明

- **Prompt 组装**：系统指令（角色、规则、工具列表与描述）+ 历史（对话与 Thought/Action/Observation）+ 当前用户输入；可选加入长期记忆检索结果。  
- **输出解析与路由**：正则或结构化解析 Thought/Action/Answer；若为 Action 则校验工具存在且参数符合 schema，再调工具层；若为 Answer 则返回用户并可选结束回合。  
- **工具执行与安全**：工具在沙箱或限定权限下执行；超时、异常与结果统一封装为 Observation 字符串，回填到下一轮 prompt。  
- **错误与重试**：工具失败时 Observation 含错误信息，由模型决定重试或换策略；解析失败时可回退到“请按格式重试”的提示。  
- **终止条件**：模型输出 Answer、或达到最大步数、或超时，则结束当轮并可选清理或保留状态。

---

## 三、小结与面试要点

**小结**：具备工具能力的 Agent 通常含编排层（组装 prompt、调用 LLM、解析与路由）、LLM、解析层、工具层（注册与执行）、记忆/状态；关键点包括 prompt 组装、输出解析与路由、工具执行与安全、错误回填与重试、终止条件。

**面试要点**：

- 模块：编排、LLM、解析、工具层、记忆/状态。  
- 流程：用户输入→编排组装 prompt→LLM→解析→工具执行→Observation 回填→循环或 Answer 结束。  
- 关键：工具描述注入、解析与校验、安全执行、错误回填与终止条件。

---

## 记忆要点

1. 架构：编排层 + LLM + 解析 + 工具层 + 记忆/状态。  
2. 流程：prompt 组装 → LLM → 解析 Action/Answer → 工具执行 → 观察回填 → 循环/结束。  
3. 关键：工具描述、解析校验、安全执行、错误处理、终止条件。
