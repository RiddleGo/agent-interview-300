# 第 64 题：Agent 的记忆系统设计——向量库、知识图谱、数据库

## 题目

如何设计 Agent 的记忆系统？对比向量数据库、知识图谱、传统数据库的适用场景。

---

## 一、记忆系统要解决什么？

Agent 需要**短期记忆**（当前会话的对话与工具链）和**长期记忆**（跨会话的用户偏好、历史结论、领域知识）。设计要点：存什么、怎么存、怎么检索与注入 prompt、以及如何控制长度与隐私。

---

## 二、向量数据库

**特点**：把文本/对话片段做 **embedding** 存入，按**相似度检索**（如当前 query 与历史片段的相似度），返回 top-k 再注入 prompt。

**适用**：非结构化、自然语言为主的记忆；按“语义相似”召回（如“上次用户问过类似问题”）；RAG 式长期记忆、会话摘要检索。  
**不擅**：精确键值查询、结构化关系、强一致性事务。

---

## 三、知识图谱

**特点**：存**实体与关系**（如用户-偏好-值、事件-时间-结果），按图查询（多跳、关系推理）。

**适用**：结构化关系明确的记忆（如“用户喜欢 A 不喜欢 B”“某任务依赖某工具”）；需要多跳推理时。  
**不擅**：大段自然语言、纯相似度检索；构建与维护成本高。

---

## 四、传统数据库（关系型/文档型）

**特点**：表结构或 JSON 文档，**精确查询**、事务、权限成熟。

**适用**：用户画像、配置、会话元数据、审计日志等**结构化、需精确查与事务**的记忆；工具调用记录、配额等。  
**不擅**：语义检索、自然语言片段召回。

---

## 五、对比与组合

| 存储 | 适用场景 |
|------|----------|
| **向量库** | 语义检索、非结构化片段、RAG 式长期记忆 |
| **知识图谱** | 实体关系、多跳推理、偏好与依赖 |
| **传统 DB** | 结构化、精确查、事务、审计与配置 |

**组合**：短期用内存/缓存+滑动窗口；长期用向量库做语义检索+ DB 存用户/配置+ 可选图谱存关系。

---

## 六、小结与面试要点

**小结**：记忆分短期与长期；向量库适合语义检索与片段、知识图谱适合关系与推理、传统 DB 适合结构化与事务；实际常组合使用。

**面试要点**：向量库=语义检索、非结构化；图谱=关系与多跳；DB=结构化、精确、事务；组合=短期窗口+长期向量+DB/图谱。

---

## 记忆要点

1. 向量库：语义检索、非结构化片段。  
2. 知识图谱：实体关系、多跳推理。  
3. 传统 DB：结构化、精确、事务；三者常组合。
