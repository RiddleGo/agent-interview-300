# 第 82 题：反馈收集与持续学习

## 题目

单 Agent 如何做反馈收集与持续学习？常见链路与注意点有哪些？

---

## 一、为什么要反馈与持续学习？

Agent 上线后需根据**真实使用效果**迭代：哪些意图识别错了、哪些工具调用不当、用户对回答是否满意。**反馈收集**获取这些信号；**持续学习**则把反馈用于改进模型、prompt、规则或检索等，形成闭环。二者结合才能提升长期质量与适应性。

---

## 二、反馈收集方式

**显式反馈**：用户点赞/点踩、评分、改写或纠正回复；在 UI 上提供明确入口并记录到日志（带 request_id、回复内容、反馈类型）。**隐式反馈**：用户是否继续追问、是否采纳建议（如点击推荐）、会话是否短时结束（可能不满）；需定义统计口径并与会话日志关联。**人工抽样**：对关键会话做标注（意图、正确性、满意度），用于评估与训练。**系统信号**：错误码、重试、熔断、工具失败等，可作为负向信号。收集时需**关联请求/会话**，并脱敏、合规存储。

---

## 三、持续学习链路

**离线**：定期用反馈与标注数据做**模型微调**或**检索/排序模型训练**；更新 prompt 模板或规则库；分析 badcase 归纳规则与策略。**在线**：将高质量反馈或纠错写入**经验记忆**（如 Reflexion），供后续相似请求参考；或更新**缓存与默认策略**。**评估**：用留存反馈或新抽样评估改进效果，避免负向迭代。注意**数据闭环**：从收集→标注/清洗→训练/规则更新→上线→再收集，并控制迭代周期与质量门禁。

---

## 小结与面试要点

**小结**：反馈收集包括显式（点赞/评分/纠正）、隐式（追问、采纳、会话长度）、人工抽样与系统信号；持续学习将反馈用于微调、prompt/规则更新、经验记忆与缓存，形成闭环；需关联请求、脱敏合规，并做效果评估。

**面试要点**：显式+隐式+人工+系统信号；持续学习=微调+prompt/规则+经验记忆；闭环与评估、数据合规。

---

## 记忆要点

1. 反馈：显式（点赞/评分/纠正）、隐式（追问、采纳）、人工抽样、系统信号。  
2. 持续学习：离线微调与规则更新、在线经验记忆与缓存；需效果评估与质量门禁。  
3. 注意：请求关联、脱敏合规、闭环迭代周期。
