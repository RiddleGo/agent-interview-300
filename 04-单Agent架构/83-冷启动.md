# 第 83 题：冷启动问题（新工具快速掌握）

## 题目

单 Agent 的“冷启动”指什么？新工具如何让 Agent 快速掌握？

---

## 一、冷启动指什么？

**冷启动**通常有两类含义：**新用户/新会话**缺乏历史与画像，难以做个性化与精准推荐；**新工具/新能力**刚接入，Agent 尚未积累足够示例与调优，容易误用、漏用或参数错误。本题侧重**新工具的冷启动**：如何让 Agent 在工具上线后尽快“会用、用对”。

---

## 二、新工具快速掌握的做法

**工具描述与 schema**：提供清晰、结构化的**工具说明**（名称、功能、适用场景、参数含义与示例、返回格式、常见错误）；OpenAPI/JSON Schema 等机器可读描述要完整，便于规划与参数校验。**Few-shot 与示例**：在 system 或 prompt 中加入**少量示例**（“当用户问…时，应调用工具 X，参数为…”），加速模型对齐；可从历史成功会话中抽取并去敏。**分层开放**：新工具先对**内部或小流量**开放，收集误用与失败 case，迭代描述与示例后再全量。**规则与约束**：对参数范围、必填项、依赖关系做校验与默认值，减少“格式正确但语义错误”的调用。**监控与反馈**：对新工具的调用率、成功率、错误类型做监控，结合用户反馈快速修正描述与策略。

---

## 三、与新用户冷启动的区分

新用户冷启动侧重**画像与偏好**：用默认配置、引导性问题或通用策略先服务，再通过行为与反馈逐步积累画像。新工具冷启动侧重**能力对齐**：通过描述、示例、小流量验证与监控，让 Agent 尽快正确使用新工具。二者可结合（如新用户+新工具时采用最保守的默认策略）。

---

## 小结与面试要点

**小结**：冷启动可指新用户或新工具；新工具冷启动通过完善描述与 schema、few-shot 示例、分层开放、规则校验与监控反馈，让 Agent 快速掌握并减少误用。

**面试要点**：新工具=描述+示例+小流量+校验+监控；新用户=默认配置+引导+行为积累画像。

---

## 记忆要点

1. 冷启动两类：新用户缺画像、新工具未对齐；本题侧重新工具。  
2. 新工具掌握：清晰描述与 schema、few-shot 示例、分层开放、参数校验、监控与反馈。  
3. 新用户冷启动：默认策略、引导、逐步积累画像。
