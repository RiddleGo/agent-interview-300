# 第 91 题：多 Agent 的负载均衡与动态任务分配

## 题目

如何实现多Agent的负载均衡？动态任务分配策略有哪些？

---

## 一、负载均衡的目标

让**任务**较均匀地分配到多个 Agent 或实例，避免个别过载、其余空闲，从而提升吞吐、降低延迟与排队。多 Agent 场景下“任务”可以是单次请求、子任务、或会话。

---

## 二、策略

- **轮询（Round Robin）**：按序将任务分给各 Agent；实现简单，不考虑负载与能力差异。  
- **最少连接/最少任务**：把新任务派给**当前任务数最少**的 Agent；适合任务耗时相近时。  
- **加权**：按 Agent 能力或配置给权重，按权重比例分配；能力强的多分。  
- **基于队列**：任务先进**公共队列**，多个 Worker Agent 抢任务（pull）；自然形成负载均衡，需保证任务不重复执行。  
- **能力匹配**：根据任务类型（如需要工具 A）只派给具备该能力的 Agent；结合路由表或注册表。  
- **动态反馈**：根据各 Agent 的**延迟、成功率、当前队列长度**实时调整分配；可与熔断结合，故障 Agent 少分或不分。

---

## 三、动态任务分配

“动态”指**不预先写死**谁做哪类任务，而是运行时根据负载、能力、策略决定。实现要点：**中心调度器**维护各 Agent 状态（忙/闲、队列长、能力），新任务到达时按策略选目标；或**去中心**时 Agent 主动拉取（pull）任务，通过队列与限流自然均衡。

---

## 四、小结与面试要点

**小结**：负载均衡 = 任务均匀分配；策略有轮询、最少任务、加权、队列 pull、能力匹配、动态反馈；动态分配依赖实时状态与调度或 pull 模型。

**面试要点**：能说出 2～3 种策略（如轮询、最少任务、队列）；动态分配依赖状态与调度或 pull；可与熔断结合。

---

## 记忆要点

1. 目标：任务均匀分配，提升吞吐、降低延迟。  
2. 策略：轮询、最少任务、加权、队列 pull、能力匹配、动态反馈。  
3. 动态分配：依赖实时状态与中心调度或 pull。
