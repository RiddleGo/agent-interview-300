# 第 86 题：多 Agent 协作系统——角色分工与通信机制

## 题目

设计一个多 Agent 协作系统，描述 Agent 角色分工和通信机制。

---

## 一、角色分工

多 Agent 系统通常按**角色**划分能力：**协调者/PM** 负责任务分解与分配、进度与冲突裁决；**执行者** 按专长分工（如检索 Agent、代码 Agent、审核 Agent）；**专家** 可细分领域（如法律、财务）。角色在创建时由 system prompt 与可用工具定义，运行时由协调者根据任务类型分配子目标给对应角色。

---

## 二、通信机制

- **中心化**：协调者向执行者**下发任务**（含输入、约束、截止），执行者**回传结果**（成功/失败、输出、需协助）；协调者再汇总、裁决或再分配。  
- **消息格式**：结构化消息（如 JSON）含 sender、receiver、type（task/result/request）、payload、id；便于路由、追溯与超时。  
- **通道**：可用内存队列、消息队列（如 Redis/RabbitMQ）、或 RPC/gRPC；大规模或跨进程时用 MQ 解耦与削峰。  
- **对等**：无中心时 Agent 间直接发请求/响应或广播“能力与需求”，通过契约或市场机制匹配。

---

## 三、小结与面试要点

**小结**：多 Agent 协作 = 角色分工（协调者、执行者、专家）+ 通信（中心化下发/回传或对等）；消息结构化、通道可用队列/RPC；按规模与一致性需求选中心化或对等。

**面试要点**：角色=协调者+执行者+专家；通信=任务下发与结果回传、结构化消息、队列或 RPC；中心化易管控，对等易扩展。

---

## 记忆要点

1. 角色：协调者（分解与分配）、执行者、专家。  
2. 通信：中心化下发/回传或对等；结构化消息、队列/RPC。  
3. 按规模与一致性选中心化或对等。
