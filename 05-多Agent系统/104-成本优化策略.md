# 第 104 题：多 Agent 的成本优化策略

## 题目

多Agent系统有哪些成本优化策略？（API调用、算力、存储等）

---

## 一、LLM API 成本

- **模型选择**：简单任务用**小模型/便宜模型**，复杂推理用大模型；按路由或步骤选择模型（如规划用大模型、执行用小模型）。  
- **Token 控制**：压缩 prompt（少带无关上下文、摘要历史）、限制 max_tokens、对长输出做截断或分页；缓存相同/相似请求的响应。  
- **调用次数**：减少无效重试、合并可合并的调用（如一次调用完成多步判断）；能规则/检索解决的先用规则/检索，再调 LLM。  
- **批量与异步**：非实时路径用批量 API（若有）、异步处理，摊薄单次成本与延迟敏感度。

---

## 二、算力与部署

- **扩缩容**：按负载**动态扩缩**，低峰缩容减少常驻实例；见“动态扩缩容”一题。  
- **本地/自托管**：对延迟与隐私要求高、调用量大时，可自托管小模型或专用模型，降低单次推理成本。  
- **批处理与队列**：非实时任务入队，批量调度 GPU/实例，提高利用率。

---

## 三、多 Agent 特有

- **减少 Agent 数/轮次**：能单 Agent 完成的不用多 Agent；能少一轮协作的就不多轮；避免“为了多 Agent 而多 Agent”。  
- **结果复用与缓存**：相同子任务（如相同查询的检索）结果缓存；多 Agent 共享上下文时避免重复请求相同内容。  
- **委托与剪枝**：早停——某步已能确定结果则不再派发后续子任务；只对不确定或高价值分支做深度调用。

---

## 四、存储与其它

- **存储**：日志与 trace 按保留策略归档或删除；冷数据用便宜存储。  
- **监控与预算**：按租户/项目/Agent 做**预算与配额**，超限告警或限流，防止成本失控。

---

## 五、小结与面试要点

**小结**：成本优化包括模型与 token 控制、减少调用与缓存、扩缩容与批处理、多 Agent 减轮次与结果复用、存储与预算监控；需在效果与成本间权衡。

**面试要点**：能列举 3～5 条（模型选择、token、缓存、扩缩容、少轮次）；多 Agent 强调少轮次与结果复用。

---

## 记忆要点

1. LLM：模型选择、token 控制、缓存、减少调用。  
2. 算力：扩缩容、批处理、自托管。  
3. 多 Agent：少轮次、结果复用、早停与剪枝；预算与配额。
