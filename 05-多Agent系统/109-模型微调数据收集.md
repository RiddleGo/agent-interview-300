# 第 109 题：多 Agent 场景下的模型微调数据收集

## 题目

在多Agent系统中如何收集用于模型微调的数据？

---

## 一、数据来源

- **对话与决策轨迹**：用户与 Agent 的**多轮对话**、Agent 的**推理步骤**（如 ReAct 的 thought/action/observation）、工具调用与结果；可构成 SFT（监督微调）的“输入-输出”对或更长轨迹。  
- **多 Agent 协作轨迹**：Agent 间**消息**、任务分解与汇总、冲突与共识过程；适合训练“协调”“总结”“冲突解决”等能力。  
- **人类反馈**：用户对回复的**点赞/点踩、改写、排序**；可转为偏好对（A 优于 B）用于 RLHF/DPO。  
- **成功与失败案例**：任务成功/失败的**完整 trace**（输入、中间步骤、最终结果与标签）；成功案例做正例、失败做负例或对比学习。  
- **人工标注**：对关键步骤或输出做**人工标注**（如意图、槽位、质量分数），提升监督信号质量。

---

## 二、收集要点

- **一致性**：统一 **trace_id/session_id**，保证一条轨迹完整可复现；存储时保留输入、中间状态、输出与元数据（模型、时间、Agent 角色）。  
- **脱敏与合规**：用户数据**脱敏**、去标识化；符合隐私与数据使用政策；可只存必要字段或哈希。  
- **采样与平衡**：按场景、结果（成功/失败）、用户分层**采样**，避免数据偏向单一类型；长轨迹可截断或摘要。  
- **格式**：转为模型可用的格式（如 instruction-response、多轮对话、偏好对）；统一 schema 便于后续清洗与训练。

---

## 三、多 Agent 特有

- **角色与上下文**：每条样本带**角色、上游产出**（如某 Agent 的输入来自哪条消息），便于训练“在给定上下文中生成某角色输出”。  
- **协作与共识**：收集“多 Agent 讨论→最终结论”的数据，可训练总结、仲裁或共识模型。  
- **工具与环境**：工具调用序列与结果可单独作为**工具使用**微调数据，或与对话联合建模。

---

## 四、小结与面试要点

**小结**：微调数据来自对话与决策轨迹、多 Agent 协作、人类反馈、成败案例与人工标注；收集需一致性、脱敏、采样与格式统一；多 Agent 需保留角色与协作上下文。

**面试要点**：能列举数据来源（轨迹、反馈、成败、标注）；强调 trace 完整、脱敏、多 Agent 角色与协作信息。

---

## 记忆要点

1. 来源：对话/决策轨迹、多 Agent 协作、人类反馈、成败案例、标注。  
2. 收集：trace 完整、脱敏、采样平衡、格式统一。  
3. 多 Agent：角色与上下文、协作与共识、工具序列。
