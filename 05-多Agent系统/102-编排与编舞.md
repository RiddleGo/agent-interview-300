# 第 102 题：多 Agent 的编排（Orchestration）vs 编舞（Choreography）

## 题目

解释多Agent中的编排（Orchestration）与编舞（Choreography）的区别。

---

## 一、编排（Orchestration）

**含义**：存在**中心协调者**（Orchestrator），由它**决定**何时调用谁、传什么、下一步做什么；各 Agent 主要“被调用、返回结果”，流程由中心定义。

**特点**：流程集中、易理解与修改、易监控与回滚；中心成为单点与瓶颈，扩展需考虑中心高可用与分层。

**典型**：Manager-Worker、CrewAI 的 Crew 执行、固定 DAG 的流水线。

---

## 二、编舞（Choreography）

**含义**：**无中心**流程控制器；各 Agent 根据**事件或消息**自主决定“收到什么就做什么、再发什么给谁”；整体流程由**本地规则与消息契约**“涌现”出来。

**特点**：去中心、易水平扩展、耦合低；整体行为难一眼看出、调试与一致性需依赖约定与监控。

**典型**：事件驱动、消息队列 + 多订阅者、对等 Agent 按协议响应事件。

---

## 三、对比

| 维度       | 编排                     | 编舞                     |
|------------|--------------------------|--------------------------|
| **控制**   | 中心协调者               | 分散、事件/消息驱动      |
| **流程**   | 中心定义                 | 本地规则+契约涌现        |
| **扩展**   | 中心易瓶颈               | 易水平扩展               |
| **可理解性** | 高                       | 较低                     |
| **适用**   | 复杂固定流程、强管控     | 开放、动态、高扩展       |

---

## 四、混合与选择

- **混合**：上层用编排（如总控分解任务），下层用编舞（Worker 按事件/队列消费）；或某子域编排、某子域编舞。  
- **选择**：流程固定、需强管控与可解释 → 编排；流程开放、参与方多、要扩展 → 编舞；常取混合。

---

## 五、小结与面试要点

**小结**：编排 = 中心协调者定义流程；编舞 = 无中心、事件/消息驱动、流程涌现；编排易控易理解，编舞易扩展；可按域或层级混合使用。

**面试要点**：能一句话区分两者；编排=中心定义流程，编舞=事件驱动、无中心；混合是常见实践。

---

## 记忆要点

1. 编排：中心协调者、流程集中、易控。  
2. 编舞：无中心、事件/消息、流程涌现、易扩展。  
3. 混合：上层编排、下层编舞，或按域划分。
