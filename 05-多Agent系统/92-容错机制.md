# 第 92 题：多 Agent 系统的容错机制

## 题目

设计多Agent系统的容错机制：单个Agent失败时的处理。

---

## 一、失败类型

- **单 Agent 崩溃/超时**：进程挂掉、无响应、超时未返回。  
- **返回错误或异常**：工具失败、模型报错、格式错误。  
- **质量失败**：输出不符合要求、任务未完成；需业务层判定。

---

## 二、容错策略

- **重试**：对可重试错误（超时、临时网络）限次数重试（如 2～3 次），带退避（exponential backoff）；对同一任务可换 Agent 重试。  
- **替换**：某 Agent 不可用时，由**调度层**将任务转派给同角色其他实例或备用 Agent；需服务发现与健康检查。  
- **降级**：无法完成子任务时，用**简化流程或默认结果**（如“该步骤跳过，用缓存/默认值”），保证主流程可继续。  
- **隔离与熔断**：对反复失败的 Agent 或下游做**熔断**，一段时间内不再派任务，避免雪崩；恢复后再试探放量。  
- **部分成功**：多 Agent 协作时，部分子任务失败不导致整体失败；汇总时标注“某部分不可用”或使用已有结果。  
- **状态恢复与幂等**：任务带 id、支持断点续跑或重放，避免重复执行产生副作用；关键操作设计幂等。

---

## 三、小结与面试要点

**小结**：容错针对崩溃、超时、错误与质量失败；策略包括重试、替换、降级、熔断、部分成功与状态恢复；需结合健康检查与调度。

**面试要点**：区分失败类型；说出重试、替换、降级、熔断；多 Agent 时部分成功与汇总策略。

---

## 记忆要点

1. 失败类型：崩溃/超时、错误、质量不达标。  
2. 策略：重试、替换、降级、熔断、部分成功、状态恢复。  
3. 结合健康检查与调度实现替换与熔断。
