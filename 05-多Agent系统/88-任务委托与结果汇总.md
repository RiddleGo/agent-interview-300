# 第 88 题：Agent 间的任务委托（Delegation）与结果汇总

## 题目

如何实现 Agent 间的任务委托（Delegation）和结果汇总？

---

## 一、任务委托（Delegation）

**含义**：协调者或某 Agent 将**子任务**交给其他 Agent 执行，并约定**输入、输出格式与约束**（如截止时间、质量要求）。实现要点：

- **任务描述**：结构化描述（如“对输入 X 做 Y，输出格式 Z”），便于执行者理解与校验。  
- **指派与路由**：根据任务类型、负载、能力表将任务路由到对应 Agent（或队列）；可带优先级与依赖（任务 A 完成后才派发 B）。  
- **异步与超时**：委托多为异步；设置超时与重试策略，超时未果可转派或降级。  
- **契约**：委托时可约定“仅返回结果”“或可再委托”，避免循环委托或权限扩散。

---

## 二、结果汇总

**含义**：多个 Agent 返回子结果后，由**协调者或专门模块**做聚合、去重、排序与格式统一，再生成最终输出或下一轮委托。

- **聚合策略**：简单拼接（如多段摘要拼成报告）、投票（如多 Agent 分类取多数）、合并（如多份检索结果去重排序）、或由 LLM 做“综合多份结果写一份回答”。  
- **一致性**：若结果冲突（如不同 Agent 给出不同结论），需策略：取多数、取置信度最高、或交给人/高层 Agent 裁决。  
- **溯源**：结果带来源 Agent 与任务 id，便于审计与 Bad Case 分析。  
- **部分失败**：部分子任务失败时，可用已有结果做部分汇总或显式标注“某部分不可用”，避免整体失败。

---

## 三、小结与面试要点

**小结**：任务委托 = 结构化任务描述、指派与路由、异步与超时、契约；结果汇总 = 聚合策略（拼接/投票/合并/LLM 综合）、冲突裁决、溯源与部分失败处理。

**面试要点**：委托要明确描述、路由、超时与契约；汇总要定聚合策略、冲突处理与溯源；部分失败时要有降级或标注策略。

---

## 记忆要点

1. 委托：任务描述、指派与路由、异步与超时、契约。  
2. 汇总：聚合策略、冲突裁决、溯源、部分失败处理。  
3. 实现上可结合消息队列与结果缓存。
