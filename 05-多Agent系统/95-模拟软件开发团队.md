# 第 95 题：模拟软件开发团队的多 Agent 系统（如 MetaGPT）

## 题目

设计一个模拟软件开发团队的多Agent系统（如MetaGPT）。

---

## 一、角色与流程

模拟真实团队的**角色**：**产品经理（PM）** 写 PRD、**架构师** 写设计文档与接口、**开发** 写代码、**测试** 写用例与执行、可选 **项目经理** 做任务拆解与排期。流程为**线性或带反馈**：PRD → 设计 → 开发 → 测试；某步不通过可打回上一步。

每个角色由 **Agent** 实现：独立 system prompt、可选工具（如代码执行、文件读写）、输入为上游产物的路径或内容，输出为结构化产物（Markdown、代码、测试报告）。

---

## 二、共享与传递

- **共享仓库/工作区**：所有产物写入**统一目录或版本库**（如 docs/、src/、tests/），下游角色通过路径或 API 读取上游产出；相当于“共享内存”存产物。  
- **结构化约定**：PRD、设计、代码的**格式与位置**事先约定（如 PRD.md、design.md、src/），便于解析与注入 prompt。  
- **依赖与触发**：上游产物就绪后触发下游（事件或轮询）；可选版本/版本号避免读到半成品。

---

## 三、质量与迭代

- **校验**：每步产出可做**自动校验**（如代码语法、测试通过率、PRD 与设计一致性）；不通过则重试或打回。  
- **人机介入**：关键节点（如 PRD 评审、发布前）可插入 Human-in-the-loop。  
- **迭代**：根据测试或评审反馈生成“修改建议”任务，再派给对应角色 Agent 迭代。

MetaGPT 即上述模式的典型实现：SOP 固定、角色与产物明确、共享环境传递，适合从需求到代码的自动生成与迭代。

---

## 四、小结与面试要点

**小结**：模拟开发团队 = 多角色 Agent（PM/架构/开发/测试）+ 固定流程（PRD→设计→代码→测试）+ 共享产物与触发；质量靠校验与人机介入，支持迭代；MetaGPT 为典型代表。

**面试要点**：能说出角色与流程、共享产物方式、质量与迭代；MetaGPT 以 SOP+角色+共享仓库为核心。

---

## 记忆要点

1. 角色：PM、架构师、开发、测试；流程：PRD→设计→代码→测试。  
2. 共享：统一工作区/仓库存产物；结构化约定与依赖触发。  
3. 质量：自动校验、人机介入、迭代；MetaGPT 为典型。
