# 第 96 题：多 Agent 中的竞争与合作建模

## 题目

多Agent系统中如何建模竞争（Competition）与合作（Cooperation）？

---

## 一、合作（Cooperation）

**含义**：多个 Agent **共同完成**一个目标，共享信息、分担子任务、汇总结果；整体收益依赖各方贡献。

**建模要点**：
- **目标对齐**：通过共享目标描述、奖励信号或协调者统一任务分解，使各 Agent 目标一致。
- **信息与资源共享**：通过共享内存、消息或黑板共享上下文与中间结果，避免重复与冲突。
- **任务分解与汇总**：协调者分解任务并分配，执行者回报结果，再聚合；可结合契约与接口约定。
- **激励**：在强化学习或多方博弈框架下，可设计**合作型奖励**（如团队奖励、共享回报），鼓励协作而非单方占优。

**典型**：CrewAI/MetaGPT 的流水线协作、多 Agent 共同完成一个项目。

---

## 二、竞争（Competition）

**含义**：多个 Agent **争夺**有限资源或更好结果，一方得益可能损及另一方；如竞价、选举、多智能体对抗。

**建模要点**：
- **资源与规则**：明确定义资源（如预算、槽位、权限）与分配规则（如价高者得、投票）。
- **策略与均衡**：Agent 各自优化己方策略；可引入**博弈论**（如纳什均衡、拍卖机制）分析均衡与激励兼容。
- **冲突解决**：竞争结果冲突时用规则裁决（如投票、仲裁、市场清算）。
- **与合作的混合**：如“组内合作、组间竞争”（团队间竞争、团队内合作）。

**典型**：多 Agent 竞价、辩论赛（多方观点竞争）、资源分配市场。

---

## 三、竞争与合作的混合

- **合作竞争（Coopetition）**：同一系统中既有合作（如共享基础设施、统一协议）又有竞争（如争任务、争排名）。
- **层级**：上层合作（如共同完成项目），下层竞争（如谁先抢到某子任务）；或按阶段切换（先合作调研，再竞争方案选择）。
- **机制设计**：通过**规则与激励**引导“在竞争中适度合作”（如共享部分信息以提升整体效率），避免纯竞争导致整体效率下降。

---

## 四、小结与面试要点

**小结**：合作 = 目标对齐、信息与任务共享、分解与汇总、合作型激励；竞争 = 资源与规则、策略与均衡、冲突裁决；可混合为组内合作组间竞争或按阶段切换。

**面试要点**：能区分合作与竞争的建模重点；合作强调目标与信息共享，竞争强调资源与博弈；混合场景常见，需机制设计。

---

## 记忆要点

1. 合作：目标对齐、共享、分解与汇总、合作型激励。  
2. 竞争：资源与规则、策略与均衡、冲突裁决。  
3. 混合：组内合作组间竞争、或按阶段切换；机制设计引导均衡。
