# 第 89 题：多 Agent 系统的冲突解决机制

## 题目

设计多 Agent 系统的冲突解决机制：当 Agent 意见不一致时如何处理？

---

## 一、冲突类型

- **输出冲突**：多个 Agent 对同一问题给出不同答案（如分类不同、推荐不同）。  
- **资源冲突**：多 Agent 竞争同一资源（如同一工具、同一数据写入）。  
- **目标/约束冲突**：子目标间矛盾（如 A 要快、B 要省成本）。  
- **顺序冲突**：对“谁先执行”有分歧。

---

## 二、解决策略

- **投票与多数**：对离散输出（分类、选择）用多数投票或加权投票；可结合置信度。  
- **仲裁者**：指定**协调者或专门仲裁 Agent** 根据规则或 LLM 综合各方输出做裁决；可引入“理由+结论”再裁决。  
- **优先级与规则**：事先定义优先级（如安全 > 性能 > 成本）或业务规则，冲突时按规则选。  
- **资源锁与队列**：对资源冲突用锁、队列或令牌桶，保证互斥访问。  
- **协商与多轮**：让冲突方多轮交换意见（如 AutoGen 式对话），直到达成一致或达到轮数上限后由仲裁者拍板。  
- **人机介入**：高影响冲突升级到 Human-in-the-loop，由人选择或修改结果。

---

## 三、小结与面试要点

**小结**：冲突分输出、资源、目标、顺序等；解决可依赖投票、仲裁者、优先级与规则、资源锁、多轮协商、人机介入；需事先约定策略与升级路径。

**面试要点**：能举例冲突类型；说出 2～3 种解决方式（如投票、仲裁、规则、锁）；重要场景可设计人机介入。

---

## 记忆要点

1. 冲突类型：输出、资源、目标、顺序。  
2. 策略：投票、仲裁者、优先级与规则、锁、协商、人机介入。  
3. 事先约定策略与升级路径。
